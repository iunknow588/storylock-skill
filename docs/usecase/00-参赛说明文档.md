## 2. 作品简介（Summary）

StoryLock 是一个面向个体创业者和 Agent 场景的本地授权内核 Skill 包。它把高价值登录密码、API Key、签名私钥等敏感对象放在本地加密保险库中；当外部工具、Agent 或宿主应用需要访问时，由用户在本地完成短时授权，再返回限范围的读取结果。

核心差异化：

- 不是再做一个普通密码管理器
- 不是把长期主凭据直接交给 Agent
- 而是把“本地验证 + 短时会话 + 限范围读取”组织成可复用的 Skill 包

为什么适合本次 Skill 赛道：

- 它不是单点 demo，而是有明确能力边界的可复用接口包
- 它已经形成 `SKILL.md + references + assets + runnable scripts` 的标准化迁移结构
- 它既能被说明，也能被调用，还能被演示和校验

## 3. 技术架构（Architecture）

### 3.1 当前实现架构

| 层级 | 职责 | 当前对应实现 |
|------|------|--------------|
| **Skill Package** | 统一入口、能力索引、参考文档 | `skill/src/storylock-skill-engine/SKILL.md` |
| **JS Skill-Layer** | 故事辅助、登录填充、挑战签名、强度评估 | `skill/src/storylock-skill-engine/index.js` + `assets/migrated/` |
| **Package Resources** | 迁移代码、参考文档、最小 schema/template | `assets/migrated/`、`references/`、`assets/schemas/`、`assets/templates/` |
| **Rust/WASM Build Layer** | 构建与 dist 校验，不作为当前 JS 主运行路径 | `scripts/build-wasm.mjs`、`scripts/selftest-wasm-dist.mjs`、`dist/wasm/` |

### 3.2 现阶段边界

| 项目 | 现状 |
|------|------|
| JS skill-layer | 已独立可跑 |
| demo / selftest | 已独立可跑 |
| Rust/WASM 构建 | 已接通构建与产物校验 |
| WASM host 完整接线 | 仍未完成 |
| 云备份模块 | 目前不作为本轮已实现主线 |

### 3.3 核心安全机制

| 机制 | 说明 |
|------|------|
| **分级授权模型** | 普通读取 `6-of-24` / 高权限 `12-of-24` / 恢复 `22-of-24` |
| **九宫格多选** | 每题 9 候选项，正确答案 2-4 个 |
| **短时会话** | 授权后建立限时会话，过期自动失效 |
| **按对象读取** | 每个秘密对象独立授权，不是全库一次性开放 |

## 6. 创新价值（Innovation）

### 6.1 核心创新点

| 创新点 | 说明 |
|--------|------|
| **秘密存储与访问分层** | 不仅解决存哪，还解决谁能在本地真正读出来 |
| **故事结构记忆入口** | 把长期记忆从随机字符串改为故事结构 |
| **短时、限范围授权** | Agent 不默认持有长期主凭据 |
| **九宫格替代固定密码** | 授权动作从输入固定口令改为提交选择集合 |

## 10. 参赛定位总结

StoryLock 适配本次大赛的核心依据：

1. 面向个体创业者和 Agent 场景
2. 把本地授权、短时会话、限范围读取做成可复用 Skill 包
3. 当前已形成可运行、可测试、可展示的 JS skill-layer 迁移包
4. Rust/WASM 已具备构建与产物验证入口，但还未完成完整 host 接线
5. 对评委而言，当前版本既可验证，也保留了后续 Rust 深化集成空间

**一句话定位**：

> StoryLock 是一个面向个体创业者和 Agent 场景的本地授权内核 Skill 包，用基于故事记忆的本地验证替代长期手工查找凭据，并用短时、限范围会话控制敏感对象的读取与使用。
