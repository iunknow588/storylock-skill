# StoryLock 华为云 FunctionGraph vs Vercel 对比分析

## 一、核心结论

**华为云 FunctionGraph 相比 Vercel 的优势不仅是响应速度，更在于：**

1. **合规与数据主权**：华为云在中国境内运营，符合国内数据安全法规
2. **企业级稳定性**：SLA 保障、专属技术支持、长期运营承诺
3. **生态协同**：与 OBS、APIG、KMS、CloudTable 等深度集成
4. **成本可控**：按需计费，无流量突发导致的意外账单

但 **Vercel 在开发体验和全球 CDN 方面有优势**，需根据场景权衡。

---

## 二、响应速度对比

### 2.1 冷启动延迟

| 平台 | 冷启动时间 | 优化后 | 说明 |
|------|-----------|--------|------|
| **Vercel** | 200-500ms | 100-200ms（Edge Function） | 全球 Edge 节点，就近执行 |
| **华为云 FunctionGraph** | 500-1000ms | 200-500ms（预留实例） | 需配置预留实例或预置并发 |

**分析**：
- Vercel Edge Function 冷启动更快（全球 100+ Edge 节点）
- 华为云 FunctionGraph 冷启动稍慢，但可通过**预留实例**优化至 200ms 以内
- StoryLock 场景：Agent 请求非高频实时交互，100-500ms 差异可接受

### 2.2 热请求响应

| 平台 | 热响应时间 | 说明 |
|------|-----------|------|
| **Vercel** | 10-50ms | Edge 节点缓存，全球加速 |
| **华为云 FunctionGraph** | 20-80ms | 实例复用，无额外冷启动 |

**分析**：
- 两者热响应差异不大（10-30ms），对用户体验影响微小
- StoryLock 核心逻辑在本地宿主执行，云端仅做请求转发和脱敏，响应时间占比低

### 2.3 网络延迟（中国用户）

| 平台 | 中国境内延迟 | 说明 |
|------|-------------|------|
| **Vercel** | 50-150ms | 需经过国际出口，受跨境网络影响 |
| **华为云 FunctionGraph** | 10-50ms | 境内节点，低延迟稳定 |

**分析**：
- 对于中国用户，华为云网络延迟显著优于 Vercel
- StoryLock 目标用户以中文用户为主，境内部署体验更佳

## 四、StoryLock 场景适配分析

### 4.1 架构特点决定云端要求

```
Agent 请求 -> 第三层网关（FunctionGraph/Vercel）-> 本地宿主
                ↓
            职责：请求解析、鉴权、脱敏、转发
            不执行敏感操作，不持有密钥
```

**关键洞察**：
- 第三层网关**非性能瓶颈**，核心逻辑在本地宿主
- 云端仅需**可靠转发+安全脱敏**，对延迟要求不极端
- 用户感知延迟主要来自**本地九宫格挑战**（人工操作）

### 4.2 华为云更适合 StoryLock 的原因

| 原因 | 说明 |
|------|------|
| **数据合规** | StoryLock 处理敏感授权，需符合国内数据安全法规 |
| **用户分布** | 目标用户以中文用户为主，境内部署体验更佳 |
| **生态协同** | 与 OBS（软件分发）、KMS（密钥托管）、CloudTable（审计）深度集成 |
| **长期运营** | OPC 一人公司，需稳定可靠的基础设施，避免平台政策风险 |
| **技术支持** | 企业级技术支持，问题响应更快 |

### 4.3 Vercel 仍可保留的场景

| 场景 | 说明 |
|------|------|
| **开发测试** | 快速原型验证，Git 集成一键部署 |
| **全球用户** | 若未来拓展海外用户，Vercel Edge 节点覆盖更广 |
| **静态网站** | 易安官网（yian.cdao.online）可继续托管在 Vercel，与 FunctionGraph 并存 |

## 五、混合部署建议

### 5.1 推荐架构

```
┌─────────────────────────────────────────┐
│           易安官网（yian.cdao.online）    │
│      部署：Vercel（静态网站，全球 CDN）    │
│      职责：下载入口、文档展示、状态页面     │
└─────────────────────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────┐
│        API 网关（api.yian.cdao.online）  │
│      部署：华为云 APIG + FunctionGraph   │
│      职责：请求鉴权、脱敏、转发本地宿主     │
│      原因：合规、低延迟、生态集成          │
└─────────────────────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────┐
│           本地宿主（用户设备）             │
│      职责：九宫格挑战、密钥管理、敏感执行   │
└─────────────────────────────────────────┘
```

### 5.2 分工说明

| 组件 | 部署平台 | 原因 |
|------|---------|------|
| **静态网站** | Vercel | 全球 CDN，开发体验好，免费额度充足 |
| **API 网关** | 华为云 FunctionGraph | 合规、低延迟、与 OBS/KMS/CloudTable 集成 |
| **软件分发** | 华为云 OBS | 境内 CDN，合规存储，版本管理 |
| **审计日志** | 华为云 CloudTable | 长期存储，合规分析，异常检测 |

## 六、结论

| 维度 | 结论 |
|------|------|
| **响应速度** | 华为云境内延迟更低，Vercel 全球 Edge 更快；StoryLock 场景差异可接受 |
| **核心优势** | 华为云胜在**合规、生态、稳定性**；Vercel 胜在**开发体验、全球 CDN** |
| **推荐方案** | **混合部署**：静态网站用 Vercel，API 网关用华为云 FunctionGraph |
| **长期策略** | 以华为云为主，Vercel 为辅，兼顾合规与开发效率 |

**一句话总结**：华为云 FunctionGraph 相比 Vercel 的核心优势不是响应速度，而是**合规保障、生态协同和长期稳定性**；推荐混合部署，静态网站用 Vercel，API 网关用华为云。
