易安与 StoryLock 的三个核心创新点：

**一、本地优先的 Agent 授权架构**
首创将敏感操作（签名、密码填充）从云端拉回用户本地设备的三层架构。远端 Agent 仅可发起请求，私人智能助理负责解释风险，StoryLock 本地核心完成最终授权与执行。关键边界在于：远程入口不直接访问本地核心，本地核心只返回最小必要结果，敏感执行始终保留在设备侧，避免传统方案中将私钥或密码托管给远端服务的信任风险。

**二、基于关联记忆线索的本地确认机制**
以用户个人故事作为授权验证的底层材料，替代传统固定口令或机械凭据。通过九宫格 challenge 从 24 题题集中动态抽取，用户基于自身记忆完成验证，无需记忆复杂密钥。该机制将"用户能理解、能回忆的授权线索"与"本地敏感执行"结合，降低了长期遗忘风险，同时通过对象强度策略（L1-L5）实现分级授权。

**三、可审计的递归脱敏与防重放体系**
建立完整的请求-授权-执行审计链路：EIP-712 结构化请求格式确保请求可审计；requestId/nonce/expiry 三重机制防止重放攻击；SQLite 持久化记录 challenge 状态、session 预算、失败窗口与审计日志；递归脱敏规则确保私钥、密码、答案原文等高敏字段不出本地边界。该体系使第三方 Agent 可安全调用本地能力，而无需接触任何长期秘密材料。
