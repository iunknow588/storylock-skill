## 优化建议

### 1. 🟡 建议：统一 Skill 命名规范

**问题**：`storylock-local-story-access-skill`、`storylock-local-story-processing-skill`、`storylock-remote-gateway-skill` 命名冗长，且 `storylock-skill-engine` 不是 Skill。

**建议**：
```
src/
├── skills/                           ← 统一放入 skills/ 目录
│   ├── local-story-access/
│   ├── local-story-processing/
│   └── remote-gateway/
├── engine/                           ← storylock-skill-engine 重命名
├── host/
├── shared/
├── ui/
└── yian-web/
```

**理由**：
- 消除冗余的 `storylock-` 和 `-skill` 重复前缀
- `engine/` 明确不是 Skill，避免混淆
- 符合多数项目"按类型分组"的惯例

---

### 2. 🟡 建议：将 `yian-web/public` 提升到独立目录

**问题**：`src/yian-web/public/` 是纯静态文件，放在 `src/` 下不合适。

**建议**：
```
skill/
├── web/                              ← 前端站点（从 src/yian-web 移出）
│   ├── public/
│   ├── src/                          ← 如有构建工具配置
│   └── package.json
├── src/                              ← 纯业务源码
└── web-api/                          ← 部署入口
```

**理由**：
- `src/` 应只包含需要编译/处理的源码
- 静态站点与业务逻辑分离，便于独立部署到 CDN
- 当前 `npm run build` 已将 `src/yian-web/public` 复制到 `release/web/public/`，说明已有分离意识

---

### 3. 🟡 建议：清理 `.temp/` 目录

**问题**：`.temp/dist/` 下有大量历史构建产物（如 `site-release/20260619-130336`、`windows-publish-test` 等）。

**建议**：
```
# 将 .temp/ 加入 .gitignore（确认是否已加入）
# 定期清理过期构建产物
# 考虑使用系统临时目录或 CI 缓存
```

**理由**：
- 临时文件不应长期驻留在仓库中
- 占用磁盘空间，增加克隆时间
- 当前 `.gitignore` 已包含 `.temp/`，但目录仍存在，说明是已跟踪文件

---

### 4. 🟡 建议：统一 `scripts/` 与 `src/` 下的脚本

**问题**：
- `scripts/` 根目录有 10 个子目录
- 各 Skill 内部也有 `scripts/`（如 `src/skills/remote-gateway/scripts/`）

**建议**：
```
scripts/
├── build/                    ← 合并 vercel/、release/、windows/、android/
├── test/                     ← 合并 test/、text/
├── dev/                      ← 开发工具
├── utils/                    ← 通用脚本（git/、png/）
└── runtime/                  ← 运行时脚本
```

**理由**：
- 减少顶层目录数量，降低认知负担
- 按功能聚类而非按平台分散

---

### 5. 🟢 可选：将 `web-api/` 纳入 `src/` 或创建 `deploy/` 目录

**问题**：`web-api/` 在根目录，与 `src/` 同级。

**建议方案 A**（保持现状，推荐）：
```
skill/
├── web-api/                  ← 部署入口（保持根目录）
└── src/                      ← 业务源码
```

**建议方案 B**（若多部署入口）：
```
skill/
├── deploy/                   ← 部署配置目录
│   ├── web-api/
│   ├── serverless/
│   └── edge/
└── src/
```

**理由**：
- 当前只有一个部署入口，保持根目录更简洁
- 若未来增加 serverless/edge 部署，再考虑 `deploy/` 目录

---

### 6. 🟡 建议：优化 `release/` 结构

**问题**：`release/web/public/` 是构建产物，但 `release/app/` 下似乎也有源码。

**建议**：
```
release/                      ← 纯构建产物（.gitignore）
├── web/
├── android/
├── windows/
└── linux/

# 将 app 源码移回 src/
src/
└── app/
    ├── android/
    └── windows/
```

**理由**：
- `release/` 应只包含构建产物，不应有源码
- 当前 `src/host/` 已有 Android/Windows host 源码，需确认 `release/app/` 是否为重复

---

### 7. 🟡 建议：文档结构优化

**问题**：`docs/` 下有 `design/`、`management/`、`ref/`、`test/`，但 `_build/` 是生成产物。

**建议**：
```
docs/
├── design/                   ← 设计文档
├── management/               ← 管理文档
├── ref/                      ← 参考文档
├── test/                     ← 测试文档
└── .build/                   ← 生成产物（改为隐藏目录，加入 .gitignore）
```

---

## 优先级汇总

| 优先级 | 建议 | 影响 | 工作量 |
|--------|------|------|--------|
| 🔴 高 | 清理 `.temp/` 目录 | 减少仓库体积 | 低 |
| 🔴 高 | 确认 `release/app/` 是否为源码，若是则移回 `src/` | 消除重复 | 中 |
| 🟡 中 | 统一 Skill 命名规范 | 提升可读性 | 中 |
| 🟡 中 | 将 `yian-web/public` 提升为 `web/` | 前后端分离 | 中 |
| 🟡 中 | 统一 `scripts/` 结构 | 降低认知负担 | 低 |
| 🟢 低 | 文档 `.build/` 隐藏化 | 规范目录 | 低 |
| 🟢 低 | 创建 `deploy/` 目录（未来多部署入口时） | 扩展性 | 低 |

---

## 推荐的最小化调整

若不想大动，至少做以下调整：

1. **清理 `.temp/`**：确认已加入 `.gitignore`，删除已跟踪的临时文件
2. **移动 `src/yian-web/public` → `web/public`**：更符合静态站点定位
3. **统一 Skill 目录命名**：去掉冗余前缀，如 `src/skills/local-story-access/`

```
skill/
├── web/                          ← 原 src/yian-web/public
├── web-api/                      ← 保持
├── src/
│   ├── skills/                   ← 原 storylock-*-skill
│   ├── engine/                   ← 原 storylock-skill-engine
│   ├── host/
│   ├── shared/
│   └── ui/
├── scripts/
├── docs/
├── release/                      ← 纯构建产物
└── .tools/
```
