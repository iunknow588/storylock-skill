# StoryLock 开发完善实施计划 Checklist

日期：2026-06-17

本文是 StoryLock 后续开发的唯一跟踪计划表。后续检查未完成工作时，只看本文即可；历史评审、审计和完善计划只作为背景资料，不再作为并行任务表。

当前代码基线：

- 三层主线已形成：本地故事处理、 local access 授权层、 remote gateway 包装层。
- 第二层已收敛为 object strength policy、grid verification、local authorization，不再作为故事读写层。
- 三层 e2e selftest 已通过，远程签名请求会经过本地授权并返回脱敏结果。
- 九宫格已具备题集绑定、版本字段、active/deprecated/pending 状态、high 9 格验证和错误路径自测。
- `MemorySecretStore` 已限制为开发测试用途；持久化 SQLite 生产模式不能静默使用普通内存密钥。
- EIP-712 production domain 已有拒绝 placeholder、零 chainId、零合约地址的代码校验。
