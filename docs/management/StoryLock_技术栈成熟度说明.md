# StoryLock 技术架构与工具链成熟度说明

StoryLock 采用三层隔离架构，各层边界清晰、职责明确，接口契约已完整文档化。加密体系选用 AES-256-GCM、HKDF-SHA256 与 HMAC-SHA256 等成熟标准，无自研算法风险；EIP-712 为以太坊生态标准，具备可验证性。华为云 FunctionGraph、OBS、APIG 均为成熟服务，已投入生产环境广泛使用。

技术栈与工具链成熟度如下：

**运行时与语言**

Node.js 22.x LTS 为长期支持版本，稳定性高，适合服务端开发。Rust 1.78+ 具备内存安全特性，适合密码学实现。Kotlin 1.9+ 为 Android 官方支持语言，移动开发生态成熟。

**数据库与存储**

SQLite 3.45+ 是嵌入式数据库标准，轻量可靠，适合本地数据管理。

**加密与密钥派生**

AES-256-GCM 基于 OpenSSL/BoringSSL 实现，硬件加速支持广泛，提供认证加密能力。HKDF-SHA256 遵循 RFC 5869 标准实现，无专利风险，用于密钥派生。

**华为云服务**

FunctionGraph 为 Serverless 计算平台，按需运行，无需预置服务器。OBS 是对象存储标准服务，支持全球 CDN 加速。APIG 提供 API 网关能力，统一入口管理与安全防护。

**协议标准**

EIP-712 是以太坊生态标准，支持结构化数据签名与验证，生态广泛支持。

**一句话总结**：StoryLock 技术栈全部选用成熟标准与生产级服务，无自研算法风险，工具链成熟度高，具备快速落地与长期维护能力。
