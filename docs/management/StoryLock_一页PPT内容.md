# StoryLock 一页 PPT 内容

## 标题：StoryLock — 用内涵记忆取代机械记忆

## 副标题：本地优先的 Agent 安全授权体系

---

## 核心痛点

- 密码太多记不住，记住的密码太简单
- Agent 协作需要授权，但密码不能交给 Agent
- 传统密码管理器依赖主密码，单点泄露全盘皆输
- 生物识别不可变更，一旦泄露永久失效

---

## 解决方案：故事驱动的九宫格挑战

- 用户创建私密故事（如"小时候外婆家的老槐树"）
- 系统生成 24 题题集，每次动态抽取 9 题验证
- 用户基于故事记忆回答，无需记忆复杂密码
- 内涵记忆比机械密码更自然、更持久、更安全

---

## 技术架构：本地优先三层隔离

**第一层 — 故事处理**：故事草稿、润色、题集生成、对象加密
**第二层 — 本地授权**：九宫格挑战、短时授权、防重放、审计
**第三层 — 远程网关**：请求包装、结果脱敏、设备验证

**核心原则**：敏感数据不出本地，远程仅处理脱敏结果

---

## 安全设计：混合加密 + 动态挑战

- **数据加密**：AES-256-GCM，保护故事、密码、文件
- **身份验证**：ECC P-256，验证设备签名与 Agent 请求
- **动态挑战**：每次题目组合随机生成，防止枚举攻击
- **防重放**：requestId + nonce + expiry 三重校验
- **渐进授权**：L1=6题 / L3=12题 / L4=22题，按敏感度分级

---

## 竞争优势

**传统密码管理器**：主密码泄露=全盘皆输，记忆负担重
**硬件钱包**：需购买硬件，与 Agent 协作不便
**生物识别**：不可变更，一旦泄露永久失效
**MFA**：依赖第三方服务，操作繁琐

**StoryLock**：
- 故事可更新，动态挑战防枚举
- 软件免费，无额外硬件
- 单一因子完成强认证，体验流畅
- EIP-712 签名 + 本地确认，Agent 协作安全

---

## 应用场景

- **Web3 签名**：Agent 请求交易，九宫格确认，防止恶意操作
- **Web2 密码**：Agent 自动填充，密码原文不出本地
- **文件加密**：故事加密重要文件，跨设备无缝解密
- **跨设备同步**：新设备输入故事，立即恢复全部授权

---

## 技术栈：成熟标准，无自研算法

- **加密**：AES-256-GCM + HKDF-SHA256 + HMAC-SHA256
- **签名**：ECC P-256 + EIP-712
- **平台**：华为云 FunctionGraph + OBS + APIG
- **语言**：Node.js 22 / Rust 1.78+ / Kotlin 1.9+
- **数据库**：SQLite 3.45+
