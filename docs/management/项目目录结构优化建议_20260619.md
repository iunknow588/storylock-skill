# 项目目录结构优化建议

日期：2026-06-19

## 1. 当前推荐结构

```text
skill/
  src/                      源码
    host/                   本地宿主 App 源码
      android-host/          Android 宿主工程
      windows-host/          Windows 宿主工程
    yian-web/                易安网站静态源码
    ui/                      本地预览和自测服务器
    storylock-*/             StoryLock 能力模块

  web-api/                  Web API 部署入口
  scripts/                  项目脚本
    release/                 App 打包、发布准备和上传脚本
    vercel/                  Web 部署、环境同步和预检脚本
    android/                 Android 设备闭环检查脚本
    windows/                 Windows 本地闭环检查脚本

  release/                  正式发布产物
    app/                     可上传服务器的 App 安装包来源
      android/
      windows/
      linux/
    web/public/              网站构建产物

  .temp/                    本地临时工作区，可删除、可再生成
    dist/                    发布计划、发布快照、上传计划
    runtime/                 本地运行时临时数据

  docs/                     文档
    management/              管理、决策、阶段性方案
    ref/                     对外/交付参考文档
    design/                  设计说明
    test/                    测试方案
```

## 2. 已完成的结构收敛

1. App 安装包统一放在 `release/app/`，安装和上传服务器时从这里取文件。
2. App 打包和发布脚本统一放在 `scripts/release/`，不再放在 `release/app/scripts/`。
3. Android 和 Windows 宿主源码已收进 `src/host/`。
4. Web API 入口保留在 `web-api/`，真实业务逻辑保留在 `src/skills/remote-gateway/`。
5. 网站构建产物统一输出到 `release/web/public/`。
6. 根目录 `dist/` 和 `tmp/` 已统一到 `.temp/`：
   - `.temp/dist/`
   - `.temp/runtime/`

## 3. 目录职责边界

`src/` 只放源码和源码级资源。

`scripts/` 只放可执行维护脚本，不放构建产物。

`release/` 只放正式发布产物，尤其是可上传服务器或可供下载的文件。

`.temp/` 只放临时输出、测试快照、发布计划和本地运行数据；该目录应保持可删除、可再生成。

`docs/management/` 只放项目管理、目录决策、阶段方案和复盘，不建议继续堆放零散临时记录。

## 4. 后续优化建议

1. 给 `.temp/` 增加 README，说明它可以安全删除，并列出脚本会重新生成的子目录。
2. 将 `docs/management/back/` 中仍有价值的历史结论归并到当前方案文档，过旧材料保留归档状态。
3. 统一文档中的“源码目录”和“兼容路由”表述：
   - 源码目录使用 `src/host/android-host/` 和 `src/host/windows-host/`。
   - 兼容 HTTP 路由继续使用 `/android-host/*`、`/download/android-host`、`/download/windows-host`。
4. 后续新增 Linux 宿主时，源码放 `src/host/linux-host/`，安装包放 `release/app/linux/`，打包脚本放 `scripts/release/linux/`。
5. 不建议把 `web-api/` 展开成业务目录；它应继续作为薄入口层，业务实现留在 `src/skills/remote-gateway/`。

## 5. 建议保留的根目录

```text
.git/
.tools/
.vercel/
docs/
release/
scripts/
src/
web-api/
.temp/
README.md
package.json
vercel.json
```

其中 `.tools/`、`.vercel/`、`.temp/` 属于本地或工具状态目录，不应提交其生成内容。
