# 易安与 StoryLock 整体技术架构及华为云能力应用说明（修订版 v6）

## 一、整体技术架构介绍

### 1.1 三层架构概览

易安采用本地优先的三层隔离架构，核心理念是"用内涵记忆取代机械记忆"——通过用户私密故事（九宫格题集）替代传统密码，降低记忆负担，提升安全性与便捷性。

第一层是故事处理层，运行 StoryDraftSkill、StoryRefineSkill、StrengthReviewSkill 三个核心组件。输入是用户故事素材，输出是24题题集和加密故事文件。关键边界是不读取受保护对象，不签发授权。设计理念是用内涵记忆取代机械记忆，内涵长期有效，用户可以随时更新。

第二层是本地授权层，运行 ObjectStrengthPolicySkill、GridChallengeSkill、LocalAuthorizationSkill 三个核心组件。输入是对象类型和用户答案，输出是challenge结果和短时session。关键边界是不返回长期秘密，不直接执行远程请求。设计理念是九宫格简化用户输入，提升便捷性，挑战动态随机生成。

第三层是远程网关层，运行 Web API Handler、EIP-712 Parser、Redaction Middleware 三个核心组件。输入是EIP-712结构化请求，输出是脱敏结果和审计元数据。关键边界是不持有私钥/密码，不透传敏感字段。设计理念是对 Agent 进行初步验证，采用非对称加密技术。

### 1.2 各层核心职责

第一层负责故事处理，包括故事草稿、润色、题集强度评估和对象加密。存储使用本地文件系统，存放加密故事文件和题集主档。运行环境是本地设备。核心机制是内涵长期有效，用户可以随时更新，挑战动态随机生成。

第二层负责本地授权，包括对象强度判断、九宫格验证、短时授权、防重放和审计。存储使用SQLite（8张表）。运行环境是本地设备（Android / Windows / Linux）。核心机制是九宫格简化用户输入，提升便捷性。

第三层负责远程网关，包括统一对外入口、请求包装、结果脱敏和对Agent进行初步验证。部署在华为云 FunctionGraph 和 OBS 静态托管。安全机制是采用非对称加密技术验证请求来源。

## 二、华为云技术能力应用

### 2.1 华为云技术选型总览

FunctionGraph（函数工作流）位于第三层远程网关核心计算位置，作用是Serverless运行 requestSignature 和 requestPasswordFill 两个接口。它接收来自 OpenClaw/Pharos 的 Agent 请求，经初步验证后转发至本地宿主。作为 Serverless 计算平台，它按需运行，无需预置服务器。

OBS（对象存储服务）位于第三层静态资源托管位置，作用是托管易安网站（yian-web）静态页面、APK/Windows 安装包分发。它与 FunctionGraph 配合，形成"静态站点+动态网关"完整入口。作为对象存储平台，它支持全球 CDN 加速，低成本高可用。

APIG（API 网关）位于第三层请求入口位置，作用是统一 API 路由、限流、HTTPS 终止。它将外部 Agent 请求路由至 FunctionGraph，保护后端服务。作为 API 网关平台，它提供统一入口管理和安全防护。

CloudTable / DDS 位于第三层审计日志归档位置（可选扩展），作用是长期审计日志存储与分析。它接收来自本地宿主的审计上报，支持事后追溯与合规分析。作为 NoSQL 数据库平台，它支持海量数据存储和自动扩容。

KMS（密钥管理服务）位于第三层设备公钥托管与配置加密位置（可选扩展），作用是托管已注册设备公钥、加密环境变量配置。它与本地 SecretStore 形成"设备公钥云上托管、故事密钥本地持有"的分层策略。作为密钥管理平台，它提供 HSM 硬件保护和合规认证。

### 2.2 各技术的详细说明

#### 2.2.1 FunctionGraph（函数工作流）

FunctionGraph 在第三层远程网关中承担核心计算载体角色。它 Serverless 运行 requestSignature 和 requestPasswordFill 两个接口，处理 EIP-712 请求解析、来源校验、脱敏中间件，并对 Agent 进行初步验证（采用非对称加密技术）。

外部 Agent 请求首先到达 APIG 进行路由和限流，然后转发到 FunctionGraph 进行第三层网关逻辑处理。FunctionGraph 进行初步验证（使用非对称加密技术验证请求来源），然后将请求转发到本地宿主（Windows/Android）。本地宿主执行第二层九宫格挑战和第一层故事验证，最后返回脱敏结果至 FunctionGraph，由 FunctionGraph 响应 Agent。

在项目创作开发过程中，FunctionGraph 发挥了以下作用：

Serverless 弹性方面，FunctionGraph 按需运行，无需预置服务器，适合 Agent 请求的低频突发特性。在项目开发阶段，开发者无需关心服务器运维，可专注于业务逻辑实现。

自动扩缩容方面，FunctionGraph 根据请求量自动扩展，避免资源浪费。在测试和演示阶段，可承受突发流量而不需手动调整。

按需计费方面，FunctionGraph 仅在请求处理时计费，降低 OPC（一人制公司）早期运营成本。对于创业初期资源有限的情况，可将资金集中于核心功能开发。

安全隔离方面，FunctionGraph 函数实例间隔离，单实例仅处理单个请求，降低横向攻击风险。这符合 StoryLock 安全优先的设计理念。

技术实现上，FunctionGraph 使用 Node.js 22 运行时，与本地 Skill 代码同构，降低维护成本。函数内实现 EIP-712 请求解析、来源校验、脱敏中间件、初步验证。通过 VPC 网络与本地宿主安全通信（或公网 HTTPS + 设备绑定认证）。

#### 2.2.2 OBS（对象存储服务）

OBS 在第三层承担静态资源托管与软件分发角色。它托管易安官网（yian.cdao.online）静态页面，存储 Android APK、Windows MSI/EXE 安装包，并管理软件版本与灰度发布。

OBS 托管的静态页面通过 JavaScript 调用 FunctionGraph API，实现"静态站点+动态网关"架构。下载页面展示最新版本信息，从 FunctionGraph 动态获取（版本、校验值、发布日期）。软件包上传至 OBS 后，FunctionGraph 更新版本元数据，形成闭环。

在项目创作开发过程中，OBS 发挥了以下作用：

静态网站托管方面，OBS 托管易安官网（yian.cdao.online），提供下载入口、绑定引导、状态展示。开发阶段可快速部署和更新网站内容。

软件分发方面，OBS 存储 Android APK、Windows MSI/EXE 安装包，支持全球 CDN 加速。用户可从就近节点下载，提升体验。

版本管理方面，OBS 通过版本控制实现软件包回滚与灰度发布。在迭代开发中，可快速切换版本进行 A/B 测试。

低成本高可用方面，OBS 按存储量与流量计费，适合 OPC 早期阶段。无需预置带宽，随用户增长自动扩展。

#### 2.2.3 APIG（API 网关）

APIG 在第三层承担统一入口角色。它统一路由 /api/storylock-gateway 到 FunctionGraph，进行请求限流防止暴力请求与 DDoS 攻击，并提供 HTTPS 终止统一证书管理。

APIG 将请求路由至 FunctionGraph，FunctionGraph 进行初步验证。APIG 限流策略与第二层失败锁定策略形成双层防护。APIG 日志可接入 CloudTable，用于请求分析与安全审计。

在项目创作开发过程中，APIG 发挥了以下作用：

统一路由方面，APIG 将 /api/storylock-gateway 路由至 FunctionGraph。开发者无需在函数内处理路由逻辑，聚焦业务实现。

请求限流方面，APIG 防止暴力请求与 DDoS 攻击，保护后端 FunctionGraph。在测试阶段可模拟高并发，验证限流效果。

HTTPS 终止方面，APIG 统一证书管理，本地宿主无需处理 TLS。简化开发配置，降低安全实现难度。

#### 2.2.4 CloudTable / DDS（可选扩展）

CloudTable / DDS 在第三层承担审计日志长期归档角色（可选扩展）。它进行长期审计日志存储与分析，异常访问模式检测，以及合规审计支持。

本地宿主通过 FunctionGraph 将审计日志上报至 CloudTable。CloudTable 分析结果可反馈至第二层，动态调整对象强度策略（如检测到异常则提升挑战阈值）。

在项目创作开发过程中，CloudTable / DDS 发挥了以下作用：

海量日志存储方面，本地 SQLite 审计日志定期归档至 CloudTable，释放本地空间。适合长期运行的生产环境。

合规分析方面，CloudTable 支持按 identityId、capability、时间范围查询审计记录。满足数据合规和审计要求。

异常检测方面，通过 DDS 聚合分析识别异常访问模式（如高频失败、异地登录）。可主动发现安全威胁。

#### 2.2.5 KMS（密钥管理服务，可选扩展）

KMS 在第三层承担设备公钥托管与配置加密角色（可选扩展）。它托管已注册设备的公钥，用于验证远程请求是否来自受信任设备。采用非对称加密技术验证请求来源。它加密存储第三层网关的环境变量、配置项（如数据库密码、API 密钥），防止配置泄露导致的安全风险。它与本地 SecretStore 形成"设备公钥云上托管、故事密钥本地持有"的分层安全策略。用户故事密钥长期有效，用户可以随时更新。

KMS 托管的设备公钥用于验证远程请求签名，确保请求来自已注册设备。私钥始终本地持有，从未上传云端。KMS 加密的环境变量在 FunctionGraph 启动时解密，保护配置安全。内涵长期有效，用户可以随时更新：StoryLock 采用"内涵长期有效"策略，用户故事密钥长期有效，用户可以随时更新故事内容。KMS 仅保留设备公钥管理和配置加密功能。

在项目创作开发过程中，KMS 发挥了以下作用：

设备公钥管理方面，用户更换设备时，新设备公钥上传到 KMS，旧设备公钥自动清理。用户故事内涵长期有效，仅设备标识更新。

配置加密方面，FunctionGraph 的环境变量（数据库密码、API 密钥）通过 KMS 加密存储，防止配置文件泄露导致的安全风险。

合规认证方面，华为云 KMS 通过 FIPS 140-2 Level 3 认证，满足等保 2.0/3.0 密钥管理要求。在 OPC 大赛评审中，可向评委展示企业级密钥管理安全性。

## 三、技术实现原理说明

### 3.1 核心加密体系

#### 3.1.1 密钥派生链

masterSalt 是随机 256-bit 值，存储在本地 SecretStore 中，长期有效。通过 HKDF-SHA256 派生 rootKey，即故事根密钥，不直接暴露，长期有效。rootKey 通过 HKDF-SHA256 加上 identitySalt 派生 workKey，即工作密钥，按对象类型派生，长期有效。workKey 通过 HKDF-SHA256 加上 objectNonce 派生 objectKey，即对象密钥，用于 AES-256-GCM 加密，长期有效。

技术原理方面，HKDF-SHA256 是基于 HMAC 的密钥派生函数，符合 RFC 5869，确保派生密钥的伪随机性。分层派生使得每层密钥仅用于特定目的，降低单点泄露影响面。长期有效意味着所有密钥长期有效，用户可以随时更新故事内容，符合"用内涵记忆取代机械记忆"的设计理念。

#### 3.1.2 对称加密规范

算法使用 AES-256-GCM，提供认证加密，防止篡改。密钥长度为 256-bit，符合 NIST 推荐标准。Nonce 为 96-bit 随机数，每次独立生成，同一密钥下不得复用。身份盐值使用 HMAC-SHA256(rootKey, identityId)，绑定用户身份。答案摘要使用 HMAC-SHA256(identitySalt, normalizedAnswer)，验证答案但不存储原文。

#### 3.1.3 九宫格 Challenge 机制

技术原理方面，从 24 题题集中按难度分层随机抽取 9 题（3 易 + 3 中 + 3 难）。用户基于私密故事记忆回答，答案经规范化后与派生摘要比对。答对阈值达到对象强度策略要求（L1=6, L3=12, L4=22）即通过。每次 challenge 题目组合不同，防止答案组合被枚举。

安全性分析方面，24 题题集空间意味着即使攻击者知道故事，需从 24 题中选 9 题组合，且每题答案空间巨大。答案摘要不可逆，HMAC-SHA256 输出无法逆向还原原始答案。失败锁定机制是 3 次失败锁定 15 分钟，防止暴力破解。

便捷性设计方面，九宫格简化用户输入，从 24 题中抽取 9 题，降低输入复杂度。故事记忆替代机械密码，用户记住一个故事，即可解锁全部对象。内涵长期有效，用户可以随时更新，挑战动态随机生成。

### 3.2 防重放机制

技术原理方面，requestId 使用 UUID v4，全局唯一，24 小时去重窗口。nonce 使用 uint256 递增随机数，nonce_store 记录已使用 nonce。expiry 是请求过期时间戳，默认 5 分钟，过期拒绝。三重校验意味着任一条件不满足即返回 SLG-013（REPLAY_DETECTED）。

### 3.3 递归脱敏机制

技术原理方面，定义 10 类高敏字段（私钥、密码、答案、session 密钥等）。三层间传输时按脱敏级别处理：none 级别用于内部层间，不脱敏；partial 级别用于第二层到第三层，遮盖高敏字段，保留结构；full 级别用于第三层到外部，仅返回状态/摘要，清空所有值。

### 3.4 初步验证机制（第三层）

技术原理方面，采用非对称加密技术验证请求来源。设备注册时，本地设备生成密钥对，公钥上传到 KMS，私钥本地存储。请求验证时，远程请求携带设备签名，FunctionGraph 从 KMS 获取公钥验证。设备更新时，用户更换设备，新设备公钥替换旧设备公钥（24 小时兼容期）。设备撤销时，设备丢失，管理员禁用该设备公钥。

关键设计方面，对 Agent 进行初步验证，确保请求来自受信任设备。采用非对称加密技术，私钥本地持有，公钥云端验证。内涵长期有效，用户可以随时更新故事内容。

## 四、技术成熟度评估

### 4.1 技术路线清晰度

架构设计评估为五星级，三层边界清晰，职责明确，接口契约已文档化。

加密方案评估为五星级，AES-256-GCM + HKDF-SHA256 + HMAC-SHA256 均为成熟标准，无自研算法。

协议选择评估为五星级，EIP-712 为以太坊生态标准，广泛支持，可验证。

密钥管理评估为四星级，分层派生方案成熟，但平台 SecretStore 适配仍需完善。

宿主实现评估为三星级，Windows Rust 原型完成，Android 骨架待验证，Linux 未启动。

华为云集成评估为四星级，FunctionGraph/OBS/APIG 为成熟服务，集成方案可行，待实际部署验证。

### 4.2 技术工具成熟度

Node.js 22.x LTS 成熟度高，稳定，长期支持。

Rust 1.78+ 成熟度高，内存安全，适合密码学实现。

Kotlin 1.9+ 成熟度高，Android 官方支持。

SQLite 3.45+ 成熟度高，嵌入式数据库标准。

AES-256-GCM（OpenSSL/BoringSSL）成熟度高，硬件加速支持广泛。

HKDF-SHA256（标准实现）成熟度高，RFC 5869，无专利风险。

FunctionGraph（华为云最新版）成熟度高，Serverless 成熟服务。

OBS（华为云最新版）成熟度高，对象存储标准服务。

APIG（华为云最新版）成熟度高，API 网关标准服务。

EIP-712（以太坊标准）成熟度高，生态广泛支持。

### 4.3 技术风险与缓解

Android 宿主真机验证延迟风险可能性为中，影响为高。缓解措施是优先完成 Windows 宿主闭环，Android 后续跟进。

华为云 FunctionGraph 冷启动延迟风险可能性为中，影响为中。缓解措施是预置并发实例，或采用预留实例模式。

题集强度不足被枚举风险可能性为低，影响为高。缓解措施是 StrengthReviewSkill 强制最低挑战位数，定期评估。

平台 SecretStore 兼容性问题风险可能性为中，影响为中。缓解措施是提供 MemorySecretStore 降级方案（开发模式显式启用）。

EIP-712 协议升级风险可能性为低，影响为低。缓解措施是协议版本化，支持多版本兼容。

### 4.4 技术路线演进

当前阶段目标是本地三层闭环，技术路径是 Node.js + SQLite + 平台 SecretStore，状态是已完成。

近期阶段目标是华为云部署，技术路径是 FunctionGraph + OBS + APIG 替换 Vercel，时间规划是 1-2 周。

中期阶段目标是第二层上云，技术路径是引入 ZKP，验证逻辑上云，密钥留本地，时间规划是 3-6 个月。

远期阶段目标是全栈隐私计算，技术路径是 MPC/HE 实现数据不可见、逻辑可验证，时间规划是 6-12 个月。

## 五、总结

StoryLock 的技术架构以"用内涵记忆取代机械记忆"为核心设计理念。

第一层和第二层通过九宫格简化用户输入，提升便捷性。用户记住一个故事，即可解锁全部对象。内涵长期有效，用户可以随时更新故事内容，挑战动态随机生成。

第三层对 Agent 进行初步验证，采用非对称加密技术确保请求来源可信。

华为云技术方面，FunctionGraph、OBS、APIG 在第三层提供弹性、低成本的 Serverless 运行环境。KMS 用于设备公钥管理和配置加密。

技术路线清晰，所选工具均为成熟标准（AES-256-GCM、HKDF-SHA256、EIP-712、SQLite），无自研算法风险。主要待验证项为 Android 宿主真机闭环与华为云实际部署性能调优。

一句话总结：StoryLock 以"用内涵记忆取代机械记忆"为魂，以成熟密码学标准为基，以华为云 Serverless 为翼，构建本地优先的 Agent 安全授权体系——九宫格简化输入，内涵长期有效，用户可以随时更新，挑战动态随机生成。
