## 3 分钟完整旁白稿

本视频说明 StoryLock 加密包从下载到应用的完整操作流程。整个流程只围绕一个核心原则展开：下载、设置、解锁、保存和应用，都必须指向同一个当前加密包。

首先，获取 StoryLock 包，并把它放到本地固定目录。进入后续步骤前，需要检查包目录中的关键文件，包括 `vault.stlk`、`package-manifest.json`、`resource-catalog.json` 和 `learning-policy.json`。如果这些文件不完整，应先补齐或重新获取包。

接下来打开 Host，进入 StoryLock 页面。在这里选择当前包。可以选择整个包目录，也可以直接选择包内的 `vault.stlk`。系统会自动把 `vault.stlk` 归一到所在的包根目录。设置完成后，需要确认 Host 和 StoryLock Core 使用的是同一个包路径。

如果系统提示存在多个内容不同的 `vault.stlk`，不要继续保存或应用。此时应重新选择正确的当前包，避免后续结果写入错误目录。

确认路径后，打开 StoryLock Core。Core 首先进入空模式。空模式不是错误状态，而是授权前的安全边界：它只确认当前包是谁，不加载故事详情、答案和受保护对象，因此敏感内容不会提前显示。

随后点击解锁当前包，进入九宫格授权挑战。每道题都会显示已选数量和应选数量。未完成时，系统提示未完成题号；挑战失败时，只提示题号和数量差异，不泄露正确答案。

挑战通过后，StoryLock Core 才加载当前包内容。此时可以查看并编辑故事、问题、答案和受保护对象。所有编辑都只作用于当前包，不能把临时草稿当作 Host 可直接读取的内容。

保存前，先完成学习测试和包预检。默认导出就是保存当前包，系统会把编辑结果写回同一个包目录，并保持 `vault.stlk`、`manifest`、`catalog` 和 `policy` 一致。

如果选择另存为，才会产生新的包目录。另存为不会自动替换 Host 当前包。要应用新包，必须回到 Host 手动切换路径，并重新解锁。

最后确认应用结果：Host 继续使用当前包，但只展示脱敏状态，不直接显示故事答案、原始密码、私钥或原始故事文本。到这里，StoryLock 从下载到应用的流程完成。
