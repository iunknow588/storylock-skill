# StoryLock 流程图使用说明

## 图文范围

图文材料由 [01_总览图.png](./01_总览图.png)、[02_下载流程图.png](./02_下载流程图.png)、[03_设置流程图.png](./03_设置流程图.png)、[04_使用与授权流程图.png](./04_使用与授权流程图.png)，以及 [ui-screenshots](./ui-screenshots) 目录中的界面截图和对应说明组成。

## 图中保留的内容

- 主流程步骤。
- 路径归一、空模式、挑战解锁、保存当前包等关键节点。
- 需要定位问题的关键状态信息。
- Host 与 StoryLock Core 的职责边界。

## 图中刻意不展开的内容

- `local-api`
- `capabilities`
- `call-chain`
- 原始答案、密码、私钥

这些内容属于实现内部能力或诊断信息，不适合在从下载到应用的操作手册中展开。

## 使用建议

- 对外演示或比赛答辩时，优先使用 01-04 流程图建立整体认知。
- 讲到具体操作时，切换到 `ui-screenshots` 中的真实截图和分镜说明图。
- 涉及保存、另存为、Host 应用边界时，优先使用 `22`、`25`、`26` 三张图说明，避免观众误解“保存当前包”和“另存为新包”的差异。
