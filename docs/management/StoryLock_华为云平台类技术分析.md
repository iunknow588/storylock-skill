# StoryLock 当前框架中华为云技术使用情况分析

## 一、核心结论

**当前 StoryLock 框架中，已明确使用/规划使用的华为云技术：**

| 层级 | 华为云服务 | 类型 | 状态 | 说明 |
|------|-----------|------|------|------|
| 第三层 | **FunctionGraph** | Serverless 计算 | ✅ 已规划 | 远程网关核心计算载体 |
| 第三层 | **OBS** | 对象存储 | ✅ 已规划 | 静态网站托管、软件分发 |
| 第三层 | **APIG** | API 网关 | ✅ 已规划 | 统一请求入口、限流鉴权 |
| 第三层 | **CloudTable/DDS** | NoSQL 数据库 | 🔶 可选扩展 | 审计日志长期归档 |
| 第三层 | **KMS** | 密钥管理 | 🔶 可选扩展 | 非敏感验证材料托管 |

**尚未使用但可集成的平台类华为云技术（潜在增强）：**

| 服务 | 类型 | 适用场景 | 优先级 |
|------|------|---------|--------|
| **ModelArts** | AI 平台 | 九宫格 challenge 智能评估、用户行为分析 | 中 |
| **DIS** | 数据接入 | 实时审计日志流处理 | 低 |
| **DMS** | 消息队列 | 异步请求处理、削峰填谷 | 低 |
| **CES** | 监控 | 函数性能监控、告警 | 中 |
| **LTS** | 日志服务 | 集中日志收集与分析 | 中 |

## 二、当前架构中的华为云技术详解

### 2.1 已规划使用的技术

#### FunctionGraph（函数工作流）- Serverless 计算平台

**类型**：Serverless/FaaS 平台

**在架构中的位置**：第三层远程网关核心

**作用**：
- 运行 `requestSignature`、`requestPasswordFill` 两个接口
- 处理 EIP-712 请求解析、来源校验、脱敏中间件
- 转发请求至本地宿主并返回脱敏结果

**平台特性利用**：
- **事件驱动**：HTTP 请求触发函数执行
- **自动扩缩容**：根据并发请求自动增加/减少实例
- **按需计费**：仅在请求处理时计费，闲置状态零成本
- **SnapShot 技术**：镜像冷启动从分钟级优化到秒级

**与其他模块协同**：
```
外部 Agent 请求 -> APIG（路由/鉴权）-> FunctionGraph（业务逻辑）
                                              ↓
                                    本地宿主（第二层/第一层）
                                              ↓
                                    返回脱敏结果 -> FunctionGraph -> 响应
```

#### OBS（对象存储服务）- 存储平台

**类型**：对象存储平台

**在架构中的位置**：第三层静态资源托管

**作用**：
- 托管易安官网静态页面（HTML/CSS/JS）
- 存储 Android APK、Windows 安装包
- 软件版本管理与灰度发布

**平台特性利用**：
- **静态网站托管**：直接作为网站入口
- **全球 CDN**：就近分发，提升下载速度
- **版本控制**：支持对象版本管理，便于回滚

**与其他模块协同**：
- OBS 托管页面通过 JavaScript 调用 FunctionGraph API
- 下载页面从 FunctionGraph 动态获取版本元数据

#### APIG（API 网关）- 网关平台

**类型**：API 管理平台

**在架构中的位置**：第三层统一入口

**作用**：
- 统一路由：`/api/storylock-gateway` -> FunctionGraph
- 请求鉴权：EIP-712 签名验证、Agent 身份校验
- 限流防护：防止暴力请求与 DDoS
- HTTPS 终止：统一证书管理

**平台特性利用**：
- **流量控制**：按 identity 限流（100/min/用户）
- **访问控制**：白名单、黑名单、签名验证
- **监控告警**：请求量、延迟、错误率统计

**与其他模块协同**：
- APIG 鉴权通过后转发至 FunctionGraph
- APIG 限流与第二层失败锁定形成双层防护

### 2.2 可选扩展的平台类技术

#### CloudTable / DDS - 数据库平台

**类型**：NoSQL 数据库平台

**适用场景**：
- 审计日志长期存储（本地 SQLite 定期归档）
- 用户-设备绑定关系存储
- 异常访问模式分析

**与当前架构的协同**：
```
本地宿主审计日志 -> FunctionGraph -> CloudTable（分区存储）
                                              ↓
                                    DDS 聚合分析异常模式
                                              ↓
                                    反馈至第二层调整策略
```

#### KMS（密钥管理服务）- 安全平台

**类型**：密钥/凭证管理平台

**适用场景**：
- 托管 EIP-712 domain 公钥（可公开验证材料）
- 加密存储第三层环境变量、配置项
- 与本地 SecretStore 形成"公开材料云上、敏感密钥本地"分层策略

**与当前架构的协同**：
```
KMS 托管公钥 -> FunctionGraph 验证 EIP-712 签名
KMS 加密配置 -> FunctionGraph 启动时解密
本地 SecretStore -> 持有私钥/密码等敏感材料
```

## 三、尚未使用但可增强架构的平台类技术

### 3.1 ModelArts（AI 开发平台）

**类型**：AI/ML 平台

**潜在应用场景**：

| 场景 | 说明 | 实现方式 |
|------|------|---------|
| **智能 challenge 评估** | 分析用户答题模式，动态调整题目难度 | 训练用户行为模型，识别异常答题模式 |
| **题集强度自动评估** | 基于大模型评估故事题集的安全性 | 调用预训练 NLP 模型分析故事熵值 |
| **异常检测** | 识别非人类操作模式（如自动化攻击） | 实时分析请求特征，标记异常 |

**与当前架构集成**：
```
用户答题数据 -> FunctionGraph -> ModelArts（推理服务）
                                      ↓
                            返回风险评估分数 -> 调整 challenge 阈值
```

### 3.2 DIS（数据接入服务）- 流处理平台

**类型**：实时数据流平台

**潜在应用场景**：
- 实时审计日志流：本地宿主日志 -> DIS -> FunctionGraph 实时处理
- 事件驱动架构：文件上传、设备绑定等事件触发函数执行

### 3.3 DMS（分布式消息服务）- 消息队列平台

**类型**：消息中间件平台

**潜在应用场景**：
- 异步请求处理：高并发时削峰填谷
- 任务队列：批量审计日志上报、密钥轮换任务

### 3.4 CES（云监控服务）- 监控平台

**类型**：监控告警平台

**潜在应用场景**：
- FunctionGraph 性能监控：调用次数、延迟、错误率、内存使用
- 自定义指标：challenge 成功率、session 颁发数量、异常请求占比
- 告警通知：异常模式触发短信/邮件告警

### 3.5 LTS（日志服务）- 日志平台

**类型**：日志收集分析平台

**潜在应用场景**：
- 集中收集 FunctionGraph、APIG、本地宿主日志
- 日志检索：按 identityId、时间范围、操作类型查询
- 日志分析：识别攻击模式、性能瓶颈

## 七、结论

| 维度 | 现状 | 建议 |
|------|------|------|
| **已使用平台类技术** | 3 项（FunctionGraph、OBS、APIG） | 已覆盖核心计算、存储、网络 |
| **可选扩展** | 2 项（CloudTable、KMS） | 建议中期集成，增强安全与审计 |
| **潜在增强** | 5 项（ModelArts、DIS、DMS、CES、LTS） | 按优先级逐步引入 |
| **平台覆盖度** | 计算/存储/网络已覆盖，安全/AI/运维待增强 | 短期补齐监控日志，中期扩展安全数据库 |

**一句话总结**：当前框架已使用 FunctionGraph、OBS、APIG 三个核心平台类技术，覆盖计算、存储、网络；建议短期补齐 CES/LTS 监控运维，中期集成 CloudTable/KMS 增强安全审计能力。
