# 易安与 StoryLock 作品简介及应用场景

## 一、作品简介

### 1.1 项目背景

数字化个人代理（Agent）正在将传统的"用户亲自授权"转向"Agent 自主发起请求"。这提高了自动化效率，也把更多敏感能力暴露给远端服务环境——签名私钥、账户密码、文件密钥等核心资产面临被中心化服务托管的风险。

对于强调用户自主管理资产、身份和权限的去中心化方向而言，这种信任集中化趋势是一种技术倒退。市场亟需一种机制，把敏感操作从云端拉回用户可控的本地设备，同时保留 Agent 协作的便利性。

易安与 StoryLock 正是基于这一痛点而生：远端 Agent 可以提出请求，私人智能助理解释风险，StoryLock 本地核心完成授权与敏感执行，用户始终在本地设备上掌握最终确认权。

### 1.2 作品主要特点

| 特点 | 说明 |
|------|------|
| **本地优先** | 签名、密码填充、文件密钥管理等敏感操作完全在本地设备执行，远端不接触密钥材料 |
| **故事化验证** | 用户以精心设计的私密故事概况替代传统固定符号口令，通过九宫格动态 challenge 完成授权确认 |
| **三层隔离** | 第一层处理文件加密与操作授权，第二层封装底层操作与风险评估，第三层作为远程网关统一对外入口 |
| **动态挑战** | 24 题题集随机组合生成九宫格 challenge，每次验证题目不同，避免固定口令的泄露风险 |
| **终身免轮换** | 单故事终身制，通过题集渐进式淘汰实现安全更新，无需用户定期更换记忆内容 |
| **可审计脱敏** | EIP-712 结构化请求、requestId/nonce/expiry 防重放、SQLite 审计日志、递归脱敏确保第三方 Agent 无法接触秘密材料 |

## 二、作品应用场景

### 2.1 核心场景：Web3 签名授权

**问题**：DeFi 用户将私钥导入 MetaMask 等热钱包，或授权给远端交易机器人，面临私钥泄露、恶意签名、钓鱼攻击等风险。

**解决方案**：
- 用户私钥由 StoryLock 本地核心管理，存储于设备安全区域（Android Keystore / Windows DPAPI）
- 交易签名请求经易安远程入口送达本地，私人智能助理展示交易详情（金额、目标合约、Gas 费）
- 用户通过九宫格 challenge 确认身份，StoryLock 本地完成签名，仅返回签名结果上链
- 远端 Agent 从未接触私钥，钓鱼攻击无法窃取签名能力

**受众**：DeFi 交易者、NFT 持有者、DAO 参与者

### 2.2 核心场景：Web2 密码填充

**问题**：用户在不同平台使用相同或弱密码，密码管理器（如 1Password、Bitwarden）将主密码或加密库托管于云端，存在单点泄露风险。

**解决方案**：
- 用户密码由 StoryLock 本地核心加密存储，密钥由私密故事派生
- 登录请求经易安远程入口转达本地，私人智能助理展示目标站点、请求来源
- 用户确认后，StoryLock 自动填充密码，远端仅接收登录结果而非密码明文
- 即使云端服务被攻破，攻击者无法获取密码库或解密密钥

**受众**：高频登录用户、多账户管理者、企业运维人员

### 2.3 核心场景：本地文件加密与授权

**问题**：用户敏感文件（病历、合同、私密照片）存储于云盘，依赖平台加密，存在平台侧泄露、审查、数据挖掘风险。

**解决方案**：
- 文件由 StoryLock 本地核心使用 AES-256-GCM 加密，密钥由私密故事派生
- 文件可上传至任意云存储，但内容为加密状态，平台无法解密
- 授权他人访问时，通过九宫格 challenge 确认，生成限时解密会话
- 访问记录完整审计，可随时撤销授权

**受众**：隐私敏感用户、自由职业者、小型企业主

### 2.4 扩展场景：视频自动化发布

**问题**：内容创作者将视频上传至中心化平台，面临版权争议、平台算法审查、收益分配不透明等问题。

**解决方案**：
- 视频文件由 StoryLock 本地签名，生成不可篡改的创作时间戳与权属证明
- 通过易安远程入口分发至多个平台，签名验证确保内容未被篡改
- 创作者保留本地密钥，平台仅验证签名而非托管版权

**受众**：独立创作者、自媒体运营者、数字艺术家

### 2.5 扩展场景：物联网设备密钥管理

**问题**：智能家居设备（摄像头、门锁、音箱）的配对密钥由厂商云端管理，存在厂商侧泄露、远程操控、数据滥用风险。

**解决方案**：
- 设备配对密钥由 StoryLock 本地核心生成与管理，厂商云端不持有密钥
- 设备控制请求经本地确认后执行，用户可随时撤销设备授权
- 即使厂商服务中断，用户仍可通过本地核心控制设备

**受众**：智能家居用户、物联网设备厂商、隐私敏感家庭

---

## 三、总结

易安与 StoryLock 的核心价值在于**重建用户对敏感数字资产的控制权**。在 Agent 自动化趋势下，传统方案要求用户将信任让渡给远端服务，而 StoryLock 通过本地优先架构、故事化验证、动态挑战机制，让用户在享受自动化便利的同时，始终掌握密钥与授权的终极决策权。

**一句话定位**：StoryLock 是 Agent 时代的个人数字保险箱——远端可以请求，但打开保险箱的钥匙，永远只在用户手中。
