# StoryLock 中 KMS 密钥管理范围与边界分析

## 一、核心结论

**KMS 在 StoryLock 中仅管理"可公开验证材料"，不管理第二层核心密钥。**

| 密钥类型 | 存储位置 | 原因 | 示例 |
|---------|---------|------|------|
| **第二层核心密钥**（masterSalt、rootKey、workKey、objectKey） | **本地 SecretStore**（操作系统密钥链/DPAPI/Android Keystore） | 敏感操作本地化，符合"本地优先"设计哲学 | 故事加密密钥、题集派生密钥 |
| **第二层会话密钥**（sessionKey） | **本地内存**（不持久化） | 短时有效，过期即销毁 | 九宫格 challenge 通过后颁发的临时授权密钥 |
| **第三层验证材料**（EIP-712 domain 公钥、证书） | **华为云 KMS** | 可公开验证，无需保密 | 验证 Agent 请求签名的公钥 |
| **第三层配置密钥**（API 密钥、环境变量加密密钥） | **华为云 KMS** | 保护配置安全，防止泄露 | FunctionGraph 环境变量加密 |

**一句话：KMS 管理的是"验证别人"的公开材料，第二层管理的是"保护自己"的私密密钥。**

## 二、KMS 具体管理范围

### 2.1 明确由 KMS 托管的材料

```
┌─────────────────────────────────────────┐
│  华为云 KMS（密钥管理服务）              │
│  ├─ EIP-712 Domain 公钥                  │
│  ├─ 证书（X.509）                        │
│  ├─ 合约地址（链上验证）                  │
│  ├─ 环境变量加密密钥（DEK）               │
│  └─ 配置项加密密钥                         │
└─────────────────────────────────────────┘
              ↓
        可公开验证 / 保护配置安全
```

#### 1. EIP-712 Domain 公钥

**用途**：验证 Agent 提交的 EIP-712 结构化请求签名

**为什么可以公开**：
- 公钥天然可公开，用于验证而非保密
- 即使泄露，攻击者也无法伪造签名（无对应私钥）
- 符合"不信任任何第三方"原则：任何人都可以验证，但无法伪造

**工作流程**：
```
Agent 请求（含 EIP-712 签名）
        ↓
FunctionGraph 从 KMS 获取公钥
        ↓
验证签名有效性
        ↓
签名有效 → 继续处理
签名无效 → 返回 SLG-001（身份验证失败）
```

#### 2. 证书（X.509）

**用途**：HTTPS 通信证书、代码签名证书

**为什么可以公开**：
- 证书本身就是公开分发的
- 用于建立信任链，而非保密

#### 3. 环境变量加密密钥（DEK）

**用途**：加密 FunctionGraph 的环境变量、配置项

**为什么由 KMS 管理**：
- 配置项包含数据库连接串、API 密钥等敏感信息
- KMS 提供硬件级加密（HSM），安全性高于软件加密
- 函数启动时通过 KMS 解密，运行时不暴露明文

**工作流程**：
```
FunctionGraph 启动
        ↓
从 KMS 获取 DEK
        ↓
解密环境变量（数据库密码、API 密钥等）
        ↓
函数运行期间使用明文
        ↓
函数销毁，内存清零
```

### 2.2 明确不由 KMS 管理的密钥

```
┌─────────────────────────────────────────┐
│  本地 SecretStore（操作系统密钥链）       │
│  ├─ masterSalt（故事根盐值）              │
│  ├─ rootKey（故事根密钥）                 │
│  ├─ workKey（工作密钥，按对象类型派生）     │
│  ├─ objectKey（对象密钥）                  │
│  └─ 私钥（签名用，如 Ed25519）              │
└─────────────────────────────────────────┘
              ↓
        绝对敏感，不出本地
```

#### 1. masterSalt（故事根盐值）

**用途**：派生 rootKey 的初始盐值，随机生成，终身不变

**为什么必须本地**：
- 一旦泄露，所有故事密钥可被重新派生
- 属于"单点失效"的最高敏感级别
- 符合 StoryLock "单故事终身制"设计

#### 2. rootKey（故事根密钥）

**用途**：通过 HKDF-SHA256 派生 workKey

**为什么必须本地**：
- 直接用于加密故事文件和题集
- 泄露意味着故事内容可被解密

#### 3. workKey / objectKey（工作/对象密钥）

**用途**：AES-256-GCM 加密的具体密钥

**为什么必须本地**：
- 直接参与加密运算
- 即使 workKey 泄露，也仅能影响单个对象（分层设计降低影响面）

#### 4. 私钥（签名用）

**用途**：用户身份签名、设备绑定证明

**为什么必须本地**：
- 私钥是身份的唯一证明
- 泄露意味着身份可被伪造

## 三、KMS 与第二层的连接关系

### 3.1 不是"直接连接"，而是"分层验证"

```
┌─────────────────────────────────────────┐
│  第三层：远程网关（华为云）                │
│  ├─ FunctionGraph                        │
│  ├─ KMS（托管公钥/证书/DEK）              │
│  └─ APIG（统一入口）                      │
└──────────────────┬──────────────────────┘
                   │  HTTPS（脱敏传输）
                   ▼
┌─────────────────────────────────────────┐
│  第二层：本地授权（用户设备）              │
│  ├─ SecretStore（操作系统密钥链）          │
│  │   ├─ masterSalt                       │
│  │   ├─ rootKey                          │
│  │   ├─ workKey                          │
│  │   └─ 私钥                             │
│  ├─ SQLite（授权数据库）                  │
│  └─ 九宫格 Challenge 引擎                 │
└──────────────────┬──────────────────────┘
                   │  本地进程调用
                   ▼
┌─────────────────────────────────────────┐
│  第一层：故事处理（用户设备）              │
│  ├─ 加密故事文件                          │
│  └─ 题集主档（加密存储）                   │
└─────────────────────────────────────────┘
```

### 3.2 三层之间的密钥流转

| 流转方向 | 流转内容 | 密钥类型 | 说明 |
|---------|---------|---------|------|
| 第一层 → 第二层 | 题集答案摘要 | HMAC-SHA256 输出 | 不可逆，仅用于验证 |
| 第二层 → 第三层 | 短时 session 令牌 | 派生密钥 | 3-30 分钟有效期，过期失效 |
| 第三层 → 第二层 | 请求签名 | EIP-712 签名 | 需用 KMS 公钥验证 |
| 第三层 → 外部 | 脱敏结果 | 无密钥 | 仅返回状态/摘要 |

**关键原则**：
- 第二层到第三层：仅传递**验证结果**（session 令牌），不传递**密钥本身**
- 第三层到第二层：仅传递**请求签名**，请求第二层验证
- 任何层级都不直接传递 masterSalt、rootKey、私钥等核心密钥

## 四、KMS 与第二层的具体交互场景

### 4.1 场景一：Agent 请求签名验证

```
Agent（外部）
  ↓ 发送 EIP-712 签名请求
FunctionGraph（第三层）
  ↓ 从 KMS 获取公钥
KMS（华为云）
  ↓ 返回公钥
FunctionGraph（第三层）
  ↓ 验证签名
  ↓ 签名有效 → 转发至本地宿主
本地宿主（第二层）
  ↓ 九宫格 Challenge 验证
  ↓ 颁发 session 令牌
FunctionGraph（第三层）
  ↓ 返回脱敏结果
Agent（外部）
```

**KMS 作用**：提供公钥用于验证，不涉及第二层密钥

### 4.2 场景二：环境变量解密

```
FunctionGraph 启动
  ↓ 从 KMS 获取 DEK（数据加密密钥）
KMS（华为云）
  ↓ 返回 DEK（需 IAM 权限验证）
FunctionGraph
  ↓ 用 DEK 解密环境变量
  ↓ 获取数据库连接串、API 密钥等
  ↓ 函数运行期间使用
  ↓ 函数销毁，内存清零
```

**KMS 作用**：保护配置安全，不涉及第二层密钥

### 4.3 场景三：证书轮换

```
证书即将过期
  ↓ 从 KMS 生成新证书
KMS（华为云）
  ↓ 返回新证书
FunctionGraph
  ↓ 更新 HTTPS 配置
  ↓ 旧证书标记过期
```

**KMS 作用**：证书生命周期管理，不涉及第二层密钥

## 五、安全边界总结

### 5.1 信任边界

| 边界 | 信任内容 | 不信任内容 |
|------|---------|-----------|
| **第二层 → 第三层** | 信任第三层转发请求 | 不信任第三层持有密钥 |
| **第三层 → 第二层** | 信任第二层验证结果 | 不信任第二层返回密钥 |
| **KMS → 第二层** | 无直接连接 | KMS 不接触第二层密钥 |
| **第二层 → KMS** | 无直接连接 | 第二层密钥不上云 |

### 5.2 泄露影响分析

| 泄露场景 | 影响范围 | 缓解措施 |
|---------|---------|---------|
| **KMS 公钥泄露** | 无实质影响（公钥本可公开） | 定期轮换 |
| **KMS DEK 泄露** | 配置项暴露（数据库密码等） | 立即轮换 DEK，重新加密配置 |
| **第二层 masterSalt 泄露** | 所有故事密钥可重新派生 | 无法缓解，需重新创建故事 |
| **第二层 rootKey 泄露** | 故事内容可解密 | 无法缓解，需重新创建故事 |
| **第二层私钥泄露** | 身份可被伪造 | 撤销设备绑定，重新注册 |

## 六、结论

| 问题 | 答案 |
|------|------|
| **KMS 管理什么？** | 可公开验证材料（公钥、证书）和保护配置安全的密钥（DEK） |
| **KMS 不管理什么？** | 第二层核心密钥（masterSalt、rootKey、workKey、objectKey、私钥） |
| **KMS 与第二层有连接吗？** | 无直接连接，通过第三层 FunctionGraph 间接交互 |
| **为什么第二层密钥不上云？** | "本地优先"设计哲学：敏感操作本地化，不信任任何第三方 |
| **KMS 泄露会影响第二层吗？** | 不会。KMS 不持有第二层密钥，两者安全边界独立 |

**一句话总结**：KMS 是第三层的"验证工具箱"和"配置保险箱"，第二层是"核心金库"。两者物理隔离，互不影响。
