# 易安与 StoryLock 技术架构及华为云能力应用说明

## 一、整体技术架构介绍

### 1.1 三层架构概览

StoryLock 采用**本地优先的三层隔离架构**，将敏感操作从云端拉回用户可控的本地设备，同时保留 Agent 协作的便利性。

```
┌─────────────────────────────────────────────────────────────┐
│  第三层：远程网关（Remote Gateway）                           │
│  职责：统一对外入口，请求包装，结果脱敏                        │
│  接口：requestSignature、requestPasswordFill                 │
│  部署：华为云 FunctionGraph / OBS 静态托管                    │
└──────────────────┬──────────────────────────────────────────┘
                   │  HTTPS / 安全通道
                   ▼
┌─────────────────────────────────────────────────────────────┐
│  第二层：本地授权（Local Authorization）                      │
│  职责：对象强度判断、九宫格验证、短时授权、防重放、审计        │
│  存储：SQLite（8张表）                                       │
│  运行：本地设备（Android / Windows / Linux）                   │
└──────────────────┬──────────────────────────────────────────┘
                   │  本地进程调用
                   ▼
┌─────────────────────────────────────────────────────────────┐
│  第一层：故事处理（Story Processing）                        │
│  职责：故事草稿、润色、题集强度评估、对象加密                  │
│  存储：本地文件系统（加密故事文件、题集主档）                  │
│  运行：本地设备                                               │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 各层核心职责

| 层级 | 核心组件 | 输入 | 输出 | 关键边界 |
|------|---------|------|------|---------|
| **第一层** | StoryDraftSkill、StoryRefineSkill、StrengthReviewSkill | 用户故事素材 | 24题题集、加密故事文件 | 不读取受保护对象；不签发授权 |
| **第二层** | ObjectStrengthPolicySkill、GridChallengeSkill、LocalAuthorizationSkill | 对象类型、用户答案 | challenge结果、短时session | 不返回长期秘密；不直接执行远程请求 |
| **第三层** | Web API Handler、EIP-712 Parser、Redaction Middleware | EIP-712结构化请求 | 脱敏结果、审计元数据 | 不持有私钥/密码；不透传敏感字段 |

## 二、华为云技术能力应用

### 2.1 华为云技术选型总览

| 华为云服务 | 在架构中的位置 | 作用说明 | 与其他模块的协同 |
|-----------|-------------|---------|----------------|
| **FunctionGraph（函数工作流）** | 第三层：远程网关 | Serverless 运行 requestSignature、requestPasswordFill 接口 | 接收来自 OpenClaw/Pharos 的 Agent 请求，经鉴权后转发至本地宿主 |
| **OBS（对象存储服务）** | 第三层：静态资源托管 | 托管易安网站（yian-web）静态页面、APK/Windows 安装包分发 | 与 FunctionGraph 配合，形成"静态站点+动态网关"完整入口 |
| **APIG（API 网关）** | 第三层：请求入口 | 统一 API 路由、限流、鉴权、HTTPS 终止 | 将外部 Agent 请求路由至 FunctionGraph，保护后端服务 |
| **CloudTable / DDS** | 第三层：审计日志归档（可选扩展） | 长期审计日志存储与分析 | 接收来自本地宿主的审计上报，支持事后追溯与合规分析 |
| **KMS（密钥管理服务）** | 第三层：非敏感密钥托管（可选扩展） | 托管 EIP-712 domain 公钥、证书等可公开验证材料 | 与本地 SecretStore 形成"公开验证材料云上托管、敏感密钥本地持有"的分层策略 |

### 2.2 各技术的详细说明

#### 2.2.1 FunctionGraph（函数工作流）

**在架构中的位置**：第三层远程网关的核心计算载体

**发挥的作用**：
- **Serverless 弹性**：按需运行，无需预置服务器，适合 Agent 请求的低频突发特性
- **自动扩缩容**：根据请求量自动扩展，避免资源浪费
- **按需计费**：仅在请求处理时计费，降低 OPC 早期运营成本
- **安全隔离**：函数实例间隔离，单实例仅处理单个请求，降低横向攻击风险

**与其他模块的协同**：
```
外部 Agent 请求 -> APIG（路由/鉴权）-> FunctionGraph（第三层网关逻辑）
                                              ↓
                                    本地宿主（Windows/Android）
                                              ↓
                                    第二层九宫格挑战 + 第一层故事验证
                                              ↓
                                    返回脱敏结果至 FunctionGraph -> 响应 Agent
```

**技术实现**：
- Node.js 22 运行时，与本地 Skill 代码同构，降低维护成本
- 函数内实现 EIP-712 请求解析、来源校验、脱敏中间件
- 通过 VPC 网络与本地宿主安全通信（或公网 HTTPS + 设备绑定认证）

#### 2.2.2 OBS（对象存储服务）

**在架构中的位置**：第三层静态资源托管与软件分发

**发挥的作用**：
- **静态网站托管**：托管易安官网（yian.cdao.online），提供下载入口、绑定引导、状态展示
- **软件分发**：存储 Android APK、Windows MSI/EXE 安装包，支持全球 CDN 加速
- **版本管理**：通过 OBS 版本控制实现软件包回滚与灰度发布
- **低成本高可用**：按存储量与流量计费，适合 OPC 早期阶段

**与其他模块的协同**：
- OBS 托管的静态页面通过 JavaScript 调用 FunctionGraph API，实现"静态站点+动态网关"架构
- 下载页面展示最新版本信息，从 FunctionGraph 动态获取（版本、校验值、发布日期）
- 软件包上传至 OBS 后，FunctionGraph 更新版本元数据，形成闭环

#### 2.2.3 APIG（API 网关）

**在架构中的位置**：第三层统一入口

**发挥的作用**：
- **统一路由**：将 `/api/storylock-gateway` 路由至 FunctionGraph
- **请求鉴权**：校验 EIP-712 签名、Agent 身份、请求来源白名单
- **限流防护**：防止暴力请求与 DDoS 攻击，保护后端 FunctionGraph
- **HTTPS 终止**：统一证书管理，本地宿主无需处理 TLS

**与其他模块的协同**：
- APIG 鉴权通过后，将请求转发至 FunctionGraph
- APIG 限流策略与第二层失败锁定策略形成双层防护
- APIG 日志可接入 CloudTable，用于请求分析与安全审计

#### 2.2.4 CloudTable / DDS（可选扩展）

**在架构中的位置**：第三层审计日志长期归档

**发挥的作用**：
- **海量日志存储**：本地 SQLite 审计日志定期归档至 CloudTable，释放本地空间
- **合规分析**：支持按 identityId、capability、时间范围查询审计记录
- **异常检测**：通过 DDS 聚合分析识别异常访问模式（如高频失败、异地登录）

**与其他模块的协同**：
- 本地宿主通过 FunctionGraph 将审计日志上报至 CloudTable
- CloudTable 分析结果可反馈至第二层，动态调整对象强度策略（如检测到异常则提升挑战阈值）

#### 2.2.5 KMS（密钥管理服务，可选扩展）

**在架构中的位置**：第三层非敏感材料托管

**发挥的作用**：
- **公开验证材料托管**：EIP-712 domain 的公钥、证书、合约地址等可公开验证材料
- **配置安全**：加密存储第三层网关的环境变量、配置项
- **与本地密钥分离**：形成"公开材料云上托管、敏感密钥本地持有"的分层安全策略

**与其他模块的协同**：
- KMS 托管的公钥用于验证 EIP-712 请求签名，私钥始终本地持有
- KMS 加密的环境变量在 FunctionGraph 启动时解密，保护配置安全

## 三、技术实现原理说明

### 3.1 核心加密体系

#### 3.1.1 密钥派生链

```
masterSalt（随机 256-bit，存平台 SecretStore）
    ↓ HKDF-SHA256
rootKey（故事根密钥，不直接暴露）
    ↓ HKDF-SHA256 + identitySalt
workKey（工作密钥，按对象类型派生）
    ↓ HKDF-SHA256 + objectNonce
objectKey（对象密钥，用于 AES-256-GCM 加密）
```

**技术原理**：
- **HKDF-SHA256**：基于 HMAC 的密钥派生函数，符合 RFC 5869，确保派生密钥的伪随机性
- **分层派生**：每层密钥仅用于特定目的，降低单点泄露影响面
- **identitySalt**：绑定用户身份，防止跨身份密钥复用攻击

#### 3.1.2 对称加密规范

| 项目 | 规范 | 说明 |
|------|------|------|
| 算法 | AES-256-GCM | 提供认证加密，防止篡改 |
| 密钥长度 | 256-bit | 符合 NIST 推荐标准 |
| nonce | 96-bit 随机数 | 每次独立生成，同一密钥下不得复用 |
| 身份盐值 | HMAC-SHA256(rootKey, identityId) | 绑定用户身份 |
| 答案摘要 | HMAC-SHA256(identitySalt, normalizedAnswer) | 验证答案但不存储原文 |

#### 3.1.3 九宫格 Challenge 机制

**技术原理**：
1. 从 24 题题集中按难度分层随机抽取 9 题（3 易 + 3 中 + 3 难）
2. 用户基于私密故事记忆回答，答案经规范化后与派生摘要比对
3. 答对阈值达到对象强度策略要求（L1=6, L3=12, L4=22）即通过
4. 每次 challenge 题目组合不同，防止答案组合被枚举

**安全性分析**：
- 24 题题集空间：即使攻击者知道故事，需从 24 题中选 9 题组合，且每题答案空间巨大
- 答案摘要不可逆：HMAC-SHA256 输出无法逆向还原原始答案
- 失败锁定：3 次失败锁定 15 分钟，防止暴力破解

### 3.2 防重放机制

**技术原理**：
- **requestId**：UUID v4，全局唯一，24 小时去重窗口
- **nonce**：uint256 递增随机数，nonce_store 记录已使用 nonce
- **expiry**：请求过期时间戳，默认 5 分钟，过期拒绝
- **三重校验**：任一条件不满足即返回 SLG-013（REPLAY_DETECTED）

**实现细节**：
```javascript
function checkReplay(requestId, nonce, expiry) {
  if (requestStore.has(requestId)) return 'SLG-013';
  if (nonceStore.has(nonce)) return 'SLG-013';
  if (Date.now() > expiry) return 'SLG-013';
  
  requestStore.set(requestId, { nonce, expiry });
  nonceStore.set(nonce, Date.now());
  return 'OK';
}
```

### 3.3 递归脱敏机制

**技术原理**：
- 定义 10 类高敏字段（私钥、密码、答案、session 密钥等）
- 三层间传输时按脱敏级别处理：
  - **none**：内部层间，不脱敏
  - **partial**：第二层→第三层，遮盖高敏字段，保留结构
  - **full**：第三层→外部，仅返回状态/摘要，清空所有值

**实现细节**：
```javascript
function redact(data, level) {
  if (level === 'full') {
    return { status: data.status, auditId: data.auditId };
  }
  if (level === 'partial') {
    return { ...data, privateKey: '***', password: '***', answer: '***' };
  }
  return data; // none
}
```

## 五、总结

StoryLock 的技术架构以**本地优先**为核心原则，通过三层隔离实现敏感操作本地化。华为云技术（FunctionGraph、OBS、APIG）在第三层远程网关中发挥关键作用，提供弹性、低成本的 Serverless 运行环境，同时与本地核心形成"敏感数据不出设备、公开服务云上托管"的安全边界。

技术路线清晰，所选工具均为成熟标准（AES-256-GCM、HKDF-SHA256、EIP-712、SQLite），无自研算法风险。主要待验证项为 Android 宿主真机闭环与华为云实际部署性能调优。

**一句话总结**：StoryLock 以成熟密码学标准为基，以华为云 Serverless 为翼，构建本地优先的 Agent 安全授权体系。
