# 易安部署与域名说明

日期：2026-06-18

本文档说明三方云服务平台上的易安远程入口在 Vercel 风格环境中的部署步骤、环境变量和同域路由检查口径。

## 1. 部署目标

目标域名：

`yian.cdao.online`

目标入口：

1. 网站首页：`https://yian.cdao.online/`
2. 网关状态与执行入口：`https://yian.cdao.online/api/storylock-gateway`
3. 易安 App 下载入口：`https://yian.cdao.online/app/download`
4. 私人智能助理绑定入口：`https://yian.cdao.online/app/bind`
5. 设备连接查看入口：`https://yian.cdao.online/app/registrations`
6. 内部兼容与 relay 入口：`https://yian.cdao.online/android-host/register`、`/android-host/relay/poll`、`/android-host/relay/respond`

## 2. Vercel 路由

当前 `vercel.json` 保持本地 UI 服务与线上部署一致：

1. `/` 指向 `src/yian-web/public/index.html`
2. `/main.js` 与 `/styles.css` 指向 `src/yian-web/public/`
3. `/assets/*` 指向 `src/yian-web/public/assets/`
4. `/api/storylock-gateway` 指向 `api/storylock-gateway.mjs`
5. `/app/download`、`/app/bind`、`/app/registrations` 指向 `api/storylock-gateway.mjs`
6. `/download/android-host` 与 `/android-host/*` 指向 `api/storylock-gateway.mjs`，作为内部工程名和旧链接兼容路径
7. `/skill-ui/*` 保留为静态 UI 兼容入口

这保证网站首页、静态资源、网关接口、下载接口与绑定接口都在同一域名下访问。普通用户页面只展示 `/app/...` 入口。

## 3. 必要环境变量

生产或候选部署至少配置：

`STORYLOCK_GATEWAY_PUBLIC_URL=https://yian.cdao.online`

`STORYLOCK_ANDROID_CONNECT_MODE=relay_url`

`STORYLOCK_ANDROID_DEEP_LINK=storylock-host://bind`

`STORYLOCK_ANDROID_REGISTRY_FILE=/tmp/storylock-android-registry.json`

APK 下载配置二选一：

`STORYLOCK_ANDROID_APK_PATH=/path/to/storylock-android-host.apk`

或：

`STORYLOCK_ANDROID_APP_DOWNLOAD_URL=https://.../storylock-android-host.apk`

推荐同时配置展示字段：

`STORYLOCK_ANDROID_APK_VERSION=0.1.0`

`STORYLOCK_ANDROID_APK_VERSION_CODE=1`

`STORYLOCK_ANDROID_PACKAGE_KIND=debug` 或 `release`

`STORYLOCK_ANDROID_RELEASE_CHANNEL=internal` 或 `candidate`

`STORYLOCK_ANDROID_APK_CHECKSUM=sha256:...`

如果启用宿主注册、relay 轮询和执行请求保护，应配置：

`STORYLOCK_ANDROID_SHARED_SECRET`

## 4. 可执行部署步骤

1. 在 Vercel 中创建或选择当前 `skill/` 对应项目。
2. 确认项目根目录为 `skill/`，并保留 `vercel.json`。
3. 配置 `STORYLOCK_GATEWAY_PUBLIC_URL=https://yian.cdao.online`。
4. 配置 Android 连接、APK 下载、版本展示和共享密钥相关环境变量。
5. 在 Vercel 项目 Domains 中绑定 `yian.cdao.online`。
6. 在 DNS 服务商处将 `yian.cdao.online` 指向 Vercel 提供的 CNAME 或 A 记录。
7. 部署后访问 `/`，确认页面样式、脚本和图片正常加载。
8. 访问 `/api/storylock-gateway`，确认返回 `onlineStatus`、`appDistribution`、`binding`、`hostRegistry`。
9. 访问 `/app/download`，确认返回 APK 文件、外部重定向或“暂未开放下载”的清晰页面。
10. 访问 `/app/bind`，确认能生成 binding token 与 deep link。
11. 访问 `/app/registrations`，确认能查看当前设备连接状态。

部署前可先执行：

```powershell
scripts\vercel\preflight.cmd -RequireDomain -RequireApkSource
```

本地联调可执行：

```powershell
scripts\vercel\preflight.cmd -BaseUrl http://127.0.0.1:4174
```

该脚本会检查必要环境变量、APK 来源、首页、静态资源、网关状态、绑定入口和下载入口。

## 5. 同域检查项

部署完成后检查：

1. `GET /` 返回首页。
2. `GET /main.js` 返回 JavaScript。
3. `GET /styles.css` 返回 CSS。
4. `GET /assets/yian-network-banner.png` 返回图片。
5. `GET /api/storylock-gateway` 返回 JSON。
6. `GET /app/download` 返回 APK、重定向或清晰的未配置页面。
7. `GET /app/bind` 返回绑定页面或 binding JSON。
8. `GET /app/registrations` 返回设备连接 JSON。
9. 网站运行态按钮可以读取网关状态。

## 6. 当前边界

域名绑定和线上环境变量写入需要在 Vercel 与 DNS 控制台完成，代码库只能提供路由、文档和自检口径。

真实 Android APK 产物、正式签名和真机闭环验证完成前，对外应使用“内测 / 候选演示”口径。

## 7. 架构边界

易安远程入口部署在三方云服务平台上，面向第三方 Agent / pharos Agent / OpenClaw 等 Skill 调用方。

推荐链路：

```text
第三方 Agent / pharos Agent / OpenClaw
  -> 通过 Skill 调用
三方云服务平台上的易安远程入口
  <-> 与用户本地设备双向通信
私人智能助理
  <-> 本地受控调用
StoryLock 本地核心（无网络服务）
```

私人智能助理可以联网并提供 AI 辅助，例如生成故事模板草稿、解释请求、提示风险；但它不直接读取故事存储服务，不持有长期密钥，也不能绕过 StoryLock 本地核心。

易安远程入口只与私人智能助理通信。StoryLock 本地核心是无网络服务，最小结果只返回给私人智能助理，再由私人智能助理按需把确认状态返回给易安远程入口。
