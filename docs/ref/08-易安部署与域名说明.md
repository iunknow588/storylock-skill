# 易安部署与域名说明

日期：2026-06-20

本文档说明当前易安网站和 StoryLock Web API 网关的部署口径、域名约定和联通检查方法。

## 1. 当前线上入口

当前评审入口：

`https://yian.cdao.online`

主要访问地址：

1. `https://yian.cdao.online/`
2. `https://yian.cdao.online/api/storylock-gateway`
3. `https://yian.cdao.online/app/download`
4. `https://yian.cdao.online/app/bind`
5. `https://yian.cdao.online/app/registrations`
6. `https://yian.cdao.online/android-host/register`
7. `https://yian.cdao.online/android-host/relay/poll`
8. `https://yian.cdao.online/android-host/relay/respond`

## 2. Vercel 路由结构

当前 `vercel.json` 主要承载：

1. `/` -> `src/yian-web/public/index.html`
2. `/main.js`、`/styles.css` -> `src/yian-web/public/`
3. `/assets/*` -> `src/yian-web/public/assets/`
4. `/api/storylock-gateway` -> `api/storylock-gateway.mjs`
5. `/app/download`、`/app/bind`、`/app/registrations` -> `/api/storylock-gateway`
6. `/app/download/android`、`/app/download/windows`、`/app/download/linux` -> `/api/storylock-gateway`
7. `/download/android-host`、`/download/windows-host`、`/download/linux-host`、`/android-host/*`、`/local-host/*` -> `/api/storylock-gateway`

其中 `/app/...`、`/download/...`、`/android-host/...` 和 `/local-host/...` 都由同一个 Web API 入口负责分发。`web-api/storylock-gateway.mjs` 是仓库内部适配层；Vercel 线上入口必须通过 `api/storylock-gateway.mjs` 暴露，否则容易出现 `404: NOT_FOUND`。

## 3. 关键环境变量

### 3.1 公开入口

`STORYLOCK_GATEWAY_PUBLIC_URL=https://yian.cdao.online`

### 3.2 Android 绑定模式

1. `STORYLOCK_ANDROID_CONNECT_MODE=relay_url`
2. `STORYLOCK_ANDROID_DEEP_LINK=storylock-host://bind`
3. `STORYLOCK_ANDROID_REGISTRY_FILE=/tmp/storylock-android-registry.json`

### 3.3 Android 下载

1. `STORYLOCK_ANDROID_APK_PATH=...`
2. `STORYLOCK_ANDROID_APP_DOWNLOAD_URL=...`
3. `STORYLOCK_ANDROID_APK_VERSION=0.1.0`
4. `STORYLOCK_ANDROID_APK_VERSION_CODE=1`
5. `STORYLOCK_ANDROID_PACKAGE_KIND=debug`
6. `STORYLOCK_ANDROID_RELEASE_CHANNEL=internal`
7. `STORYLOCK_ANDROID_APK_CHECKSUM=sha256:...`

### 3.4 宿主转发保护

`STORYLOCK_ANDROID_SHARED_SECRET`

## 4. 预检查步骤

正式部署前建议执行：

```powershell
scripts\vercel\preflight.cmd -RequireDomain -RequireApkSource
```

本地联调可执行：

```powershell
npm run build
npm run test:site-http
```

`npm run test:site-http` 会在本地启动一个等价于 Vercel rewrite 的轻量 HTTP 检查，覆盖首页、静态资源、Web API、Android/Windows/Linux 下载、绑定和注册查询。

部署到 Vercel 前还必须确认本地项目绑定：

```powershell
Get-Content .vercel\project.json
```

其中 `projectName` 必须与 `scripts\vercel\.env` 或 `scripts\vercel\.env.example` 中的 `VERCEL_PROJECT_NAME` 一致。当前仓库默认项目名为 `storylock-gateway`，用于承载 `yian.cdao.online`。如果 Vercel 控制台显示该域名绑定在其他项目，需要先确认域名归属，再执行：

```powershell
scripts\vercel\link_project.cmd
```

`scripts\vercel\preflight.ps1`、`publish_site_release.ps1` 和 `sync_env_file_to_vercel.ps1` 都会检查 `.vercel/project.json`；如果本地 link 项目与 `VERCEL_PROJECT_NAME` 不一致，会直接失败，避免把构建产物部署到错误项目。

如果本机 `.vercel/project.json` 仍显示 `projectName: "skill"`，而 `scripts\vercel\.env.example` 已显示 `VERCEL_PROJECT_NAME=storylock-gateway`，说明本地 link 仍停留在旧项目。此时即使 `vercel deploy --prod` 成功，也可能只是把产物发布到旧项目，`yian.cdao.online` 仍会返回 Vercel 的部署级 `404: NOT_FOUND`。

首次修复线上 404 时，不能让旧线上状态挡住新部署。`publish_site_release.ps1 -Preflight` 当前采用两段式检查：

1. 部署前执行本地 preflight：检查环境变量、`.vercel/project.json` 和 APK/包来源，但跳过线上 HTTP。
2. 部署前执行 `vercel whoami`，确认 Vercel CLI 已登录或 `VERCEL_TOKEN` 可用，并能访问 Vercel 认证端点。
3. `vercel deploy --yes` 成功后，再对 `STORYLOCK_GATEWAY_PUBLIC_URL` 执行线上 HTTP preflight。

如果部署时报：

```text
request to https://vercel.com/.well-known/openid-configuration failed
Client network socket disconnected before secure TLS connection was established
```

说明当前机器无法完成 Vercel CLI 的认证/OIDC 网络请求。处理顺序：

1. 在 `scripts\vercel\.env` 中配置 `VERCEL_TOKEN=...`，或先执行 `vercel login`。
2. 检查代理、防火墙、TLS 拦截或公司网络策略是否阻断 `https://vercel.com/.well-known/openid-configuration`。
3. 从可以访问 Vercel 的网络重新执行部署命令。
4. 仍失败时，可先执行 `vercel whoami`，确认 CLI 认证链路可用后再部署。

如果本机网络长期无法访问 Vercel OIDC 端点，可以改走 GitHub Actions 发布：

1. 在 GitHub 仓库 Secrets 中配置：
   - `VERCEL_TOKEN`
   - `VERCEL_ORG_ID`
   - `VERCEL_PROJECT_ID`
   - `STORYLOCK_ANDROID_SHARED_SECRET`
2. 打开 Actions，手动运行 `Yian Vercel Production Deploy`。
3. 工作流会先执行 `npm test` 和 `npm run build`，再执行 `vercel@54.5.1 deploy --prod --yes`。
4. 部署完成后，工作流会检查 `https://yian.cdao.online/`、`/main.js`、`/styles.css`、`/api/storylock-gateway` 和 `/app/download`。

也可以参考 WSL/Linux 发布方式。如果 WSL 中已有可用 `VERCEL_TOKEN` 或已登录 Vercel，可以执行：

```powershell
scripts\vercel\publish_site_release_wsl.cmd -Build -SiteHttpSmoke -Preflight -Prod -Execute
```

该脚本会在 WSL 中加载 `nvm`，选择已安装的最高 Node.js `>=22`，优先复用 `VERCEL_TOKEN`，没有全局 `vercel` 时使用 `npx --yes vercel@54.5.1`。它的思路与 `ming/scripts/deploy_vercel.sh` 一致：先确认 token/登录、项目 link 与域名可访问性，再执行非交互部署。确认域名归属无误后，可在 `scripts\vercel\.env` 设置 `VERCEL_BIND_CUSTOM_DOMAIN=true`，显式把 `yian.cdao.online` alias 到本次 production deployment。

## 5. 部署检查清单

1. 确认仓库根目录为 `skill/`。
2. 确认 `vercel.json` 已包含主站静态资源和 Web API 路由。
3. 确认 `.vercel/project.json` 的 `projectName` 与 `VERCEL_PROJECT_NAME` 一致。
4. 确认该 Vercel 项目已绑定 `yian.cdao.online`。
5. 确认 `STORYLOCK_GATEWAY_PUBLIC_URL` 与实际域名一致。
6. 确认 APK 来源已配置为本地路径或外部分发地址。
7. 确认 DNS 已正确指向 Vercel。
8. 执行 `npm run build` 和 `npm run test:site-http`。
9. 如只是检查线上当前状态，执行 `scripts\vercel\preflight.cmd -BaseUrl https://yian.cdao.online`。
10. 如要修复首次部署级 404，执行 `scripts\vercel\publish_site_release.cmd -Target vercel -Build -SiteHttpSmoke -Preflight -Prod -Execute`，让脚本先部署再做线上 HTTP preflight；如果部署返回新的 `*.vercel.app` 地址但自定义域名仍 404，确认域名属于当前项目后加 `-BindCustomDomain` 重新执行。
11. 访问 `/` 页面确认站点可打开。
12. 访问 `/api/storylock-gateway` 确认返回 JSON。
13. 访问 `/app/download` 确认下载入口有效。
14. 访问 `/app/bind` 确认可以拿到 binding token 或绑定信息。
15. 访问 `/app/registrations` 确认宿主注册状态查询可用。
16. 如线上返回 `404: NOT_FOUND`，先确认当前生产部署是否来自本仓库根目录 `skill/`，确认 `.vercel/project.json` 指向的项目就是域名绑定项目，并确认生产部署包含 `api/storylock-gateway.mjs` 与 `release/web/public/index.html`。

## 6. 联通性检查

建议最少检查以下端点：

1. `GET /`
2. `GET /main.js`
3. `GET /styles.css`
4. `GET /assets/yian-network-banner.png`
5. `GET /api/storylock-gateway`
6. `GET /app/download`
7. `GET /app/bind`
8. `GET /app/registrations`
9. `GET /app/download/android`
10. `GET /app/download/windows`
11. `GET /app/download/linux`

若启用了 Android 宿主 relay，还应检查：

1. `/android-host/register`
2. `/android-host/relay/poll`
3. `/android-host/relay/respond`

当前如果 `https://yian.cdao.online/` 返回 Vercel 的 `NOT_FOUND` 页面，应按以下顺序排查：

1. 在 Vercel 控制台确认 `yian.cdao.online` 绑定的项目名。
2. 在本地确认 `.vercel/project.json` 的 `projectName` 是否为同一个项目。
3. 确认 `VERCEL_PROJECT_NAME` 与 `.vercel/project.json` 一致。
4. 确认最新生产部署的根目录是 `skill/`，而不是仓库上级目录或其他子目录。
5. 确认生产部署使用本仓库的 `vercel.json`，其中 `outputDirectory` 为 `release/web/public`。
6. 确认生产部署包含 `api/storylock-gateway.mjs`。
7. 重新执行 `scripts\vercel\publish_site_release.cmd -Target vercel -Build -SiteHttpSmoke -Preflight -Prod -Execute`。
8. 如果本机仍因 Vercel OIDC/TLS 失败无法部署，使用 `.github/workflows/vercel-production.yml` 中的 GitHub Actions 发布通道。
9. 如果 WSL 网络或 token 可用，也可使用 `scripts\vercel\publish_site_release_wsl.cmd -Build -SiteHttpSmoke -Preflight -Prod -Execute`。

## 7. 真实说明边界

对外应准确表述：

1. 当前线上入口是易安网站和 StoryLock Web API 网关的组合部署。
2. 它负责下载、绑定、注册查询和宿主 relay。
3. 它不直接承载 StoryLock 本地核心的敏感存储和 challenge 执行。
4. 本地敏感执行仍在用户设备侧。
