# StoryLock 参赛概览

## 1. 项目概述

StoryLock 是一个本地优先的授权访问 Skill 项目。它把“基于故事记忆线索的本地验证”拆成可复用的三层能力：故事处理、访问授权、远程请求包装。

当前项目重点不是宣称已经覆盖大量应用场景，而是证明一条清晰的安全链路：

1. 本地生成或整理故事线索。
2. 根据目标对象判断所需验证强度。
3. 通过九宫格验证签发短时本地授权。
4. 将远程签名或 Web2 密码填充请求包装成可审计、可脱敏的 Skill 调用。

## 2. 当前三层结构

| 层级 | 代码包 | 职责 |
| --- | --- | --- |
| 第一层 | `src/storylock-local-story-processing-skill` | 故事草稿、故事润色、题集强度评估 |
| 第二层 | `src/storylock-local-story-access-skill` | 对象强度判断、九宫格验证、本地授权、SQLite 审计 |
| 第三层 | `src/storylock-remote-gateway-skill` | `requestSignature`、`requestPasswordFill`、远程请求包装、脱敏返回 |

`src/storylock-skill-engine` 是兼容演示包，用于展示本地密码填充和签名授权示例，不作为第四个安全层。

## 4. 适合比赛表达的核心点

StoryLock 适合作为 Skill 项目展示，原因是：

1. 能力边界清晰：远程网关不直接持有密码或私钥。
2. 输入输出结构化：签名与密码填充请求都有明确字段。
3. 本地安全约束真实存在：SQLite 记录 challenge、session、nonce、failure、audit 等状态。
4. 端到端演示可运行：第三层请求可以串联第二层授权和本地执行器。

## 5. 功能边界

当前负责：

1. 故事草稿、润色与题集强度评估。
2. 对象强度判断和九宫格验证。
3. 短时本地授权和审计落库。
4. 远程签名请求和密码填充请求的包装、委托、脱敏返回。

当前不负责：

1. 故事对象读取或写回。
2. 完整链上验证。
3. EIP-1271/ERC-4337 生产级集成。
4. 多链、多钱包、多平台账号的完整管理。
5. 把 Demo 或兼容包表述为正式安全边界本体。

## 6. 推荐参赛表述

建议统一使用下面的表述：

> StoryLock 将本地故事记忆验证、对象强度策略和远程请求包装拆成三层 Skill 能力，使 Agent 或第三方系统可以在不直接接触长期秘密的前提下，请求本地完成签名授权或 Web2 密码填充。
