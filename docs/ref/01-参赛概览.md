# StoryLock 参赛概览

## 1. 项目概述

StoryLock 是一个本地优先的授权访问内核，将“基于故事记忆的本地验证”上浮为可复用的 Skill 能力，服务于登录填充、挑战签名与故事辅助等场景。

## 2. 参赛定位

从 Skill 项目的视角看，StoryLock 并不是单纯的密码库，也不是泛化 Agent 工作流，而是一个分层清晰的本地授权系统。

| 层级 | 职责 |
|------|------|
| Core 层 | 正式授权判断 |
| Host 层 | 运行时会话与对象读取 |
| Skill 层 | 把授权访问封装为可调用能力 |
| Demo 层 | 展示能力的接入与调用方式 |

这个定位适合比赛表达，因为它能同时说明：

1. 为什么这是一个 Skill 项目
2. 为什么它不是空泛概念包装
3. 为什么它具备真实接入与演示价值

## 3. 适配 Skill 方向的核心依据

StoryLock 已具备 Skill 项目最关键的三项条件：

1. 边界清晰  
   Skill 层不是安全边界本体，而是建立在 StoryLock Core 之上的高层封装。
2. 输入输出明确  
   登录类 Skill 输出结构化字段；签名类 Skill 输出签名结果。
3. 演示路径明确  
   提供命令行 Demo、浏览器 Demo、工作台 Demo。

## 4. 当前已实现的 Skill 能力

| Skill 名称 | 功能描述 |
|-----------|---------|
| `StoryDraftAssistSkill` | 故事草稿生成辅助 |
| `StoryRefineAssistSkill` | 故事润色与提示整理 |
| `StrengthReviewSkill` | 题集强度评估 |
| `LoginAuthorizationSkill` | 登录授权访问封装 |
| `LocalPasswordFillSkill` | 本地授权后输出登录字段 |
| `SigningAuthorizationSkill` | 兼容型签名授权封装 |
| `ChallengeSigningAuthorizationSkill` | 本地授权后直接输出签名结果 |

## 5. 功能边界

负责范围：

1. 创建本地授权挑战
2. 提交授权答案并建立短时授权会话
3. 在不同授权范围内组织对象读取
4. 将授权访问结果包装为 Skill 层输出

不负责范围：

1. 改写底层 `6-of-24`、`12-of-24`、`22-of-24` 授权语义
2. 替代真实宿主 UI、部署策略或密钥治理策略
3. 替代业务系统完成全部后续动作
4. 把 Demo 或 Skill 层表述为正式安全边界本体

## 6. 推荐参赛表述

建议统一使用下面这类表述：

> StoryLock 将本地授权内核上浮为可调用的 Skill 接口，使 Agent 或宿主系统在不突破正式安全边界的前提下，获得登录填充、挑战签名与故事辅助能力。

为贴近 Pharos Skill Engine 的组织方式，当前已经从 `story-lock` 中迁移出一份可复用的 Skill 版本：

- `skill/src/storylock-skill-engine/`

该版本当前特点：

1. 已具备 `SKILL.md + references + assets` 结构
2. 已提炼出 StoryLock skill-layer 的边界说明、调用说明和 demo 说明
3. 以“迁移复用”为目标，暂不主动修改原有功能语义
4. 更适合作为比赛提交、技术附录和结构化展示版本
