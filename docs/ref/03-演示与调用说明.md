# StoryLock 演示与调用说明

## 1. 前置环境

在 `skill/` 仓库根目录确认：

```powershell
node -v
```

当前主线要求 Node.js `>=22.0.0`。

## 2. 一键运行

在 `skill/` 根目录执行：

```powershell
npm run test
```

这会完成：

1. 工作区自测
2. 文本格式检查
3. Schema 契约检查
4. 文档一致性检查
5. 路径一致性检查

如果只想先验证运行逻辑：

```powershell
node scripts/test/run-selftests.mjs
```

## 3. 三层闭环演示

在 `src/skills/remote-gateway` 执行：

```powershell
npm run selftest:e2e
```

这条链路验证：

1. 第三层 `requestSignature`
2. 第二层对象强度判断
3. 第二层九宫格验证
4. 第二层本地授权
5. 本地签名执行器
6. SQLite 审计写入
7. 第三层脱敏返回

预期输出：

```text
StoryLock three-layer e2e selftest passed.
```

## 4. Web API + Android Host Mock 演示

在 `src/skills/remote-gateway` 执行：

```powershell
npm run selftest:web-api-android
```

这条链路验证：

1. Vercel 风格 Web API 网关入口
2. Android host mock 注册
3. relay/poll/respond 闭环
4. APK/下载元数据暴露
5. 本地执行与脱敏返回

预期输出：

```text
StoryLock Web API gateway + Android registration/relay/APK selftest passed.
```

说明：

1. 这里验证的是云端入口 + 本地 host mock 的联通性。
2. 当前仓库还不是完整 Android 真机工程。
3. 对外演示时应明确这是“主线架构验证”，不是“完整移动端产品发布版”。

## 7. 题集演示命令

在 `src/skills/local-story-access` 执行：

### 7.1 生成模板

```powershell
npm run generate:question-set-template
```

### 7.2 Dry-run 校验

```powershell
npm run validate:question-set -- --input assets/question-set-master.sample.json --require-min-active 24
```

### 7.3 持久化导入

```powershell
npm run import:question-set -- --input assets/question-set-master.sample.json --db storylock.db --use-platform-secret-store
```

## 8. 清理命令

在 `src/skills/local-story-access` 执行：

### 8.1 开发内存模式

```powershell

```

### 8.2 持久化数据库 + 平台 SecretStore

```powershell
npm run cleanup -- --db E:\path\to\storylock.db --use-platform-secret-store
```

## 9. 演示时推荐讲法

推荐按下面顺序解释：

1. 第一层负责故事处理和题集强度评估
2. 第二层负责本地验证、授权、撤销、审计
3. 第三层负责远程请求包装、EIP-712 表达和脱敏返回
4. `src/engine` 只是兼容和演示包

避免以下讲法：

1. 把 `requestChallengeSign` 说成当前主线接口
2. 把第二层说成 story read/write 层
3. 把兼容演示包说成新的第四层安全边界
4. 把 Web API + Android host mock 说成完整正式移动端产品
