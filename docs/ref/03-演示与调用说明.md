# StoryLock 演示与调用说明

## 1. 最小命令行演示

在 `story-lock` 仓库根目录执行：

```powershell
npm run skill-demo
```

当前 Demo 输出三类结果：

1. 故事草稿 Skill 输出
2. 登录填充 Skill 输出
3. 挑战签名 Skill 输出

对应文件：

- `story-lock/examples/04-skill-layer-demo.mjs`

这个 Demo 主要用于证明：

1. StoryLock Skill 层已经具备真实调用面
2. 不同 Skill 的输入输出已经可独立展示
3. 本地授权与上层 Skill 包装之间的关系清晰可辨

## 2. 浏览器演示

在 `story-lock` 仓库根目录执行：

```powershell
npm run browser-demo
```

然后在浏览器打开：

1. `http://127.0.0.1:4173/browser-demo.html`
2. `http://127.0.0.1:4173/story-workbench.html`

建议分工：

| 页面 | 展示内容 |
|------|---------|
| `browser-demo.html` | 本地授权访问链路、挑战、会话与对象读取 |
| `story-workbench.html` | 工作台、模板、读取、导入导出与验证流程 |

## 3. 代码调用入口

### 3.1 包导出面

```js
import {
  StoryDraftAssistSkill,
  StoryRefineAssistSkill,
  StrengthReviewSkill,
  LocalPasswordFillSkill,
  ChallengeSigningAuthorizationSkill,
} from "story-lock/skills";
```

### 3.2 仓库内部入口

```js
import {
  StoryDraftAssistSkill,
  StoryRefineAssistSkill,
  StrengthReviewSkill,
  LocalPasswordFillSkill,
  ChallengeSigningAuthorizationSkill,
} from "../src/index.js";
```

### 3.3 迁移版 Skill 包入口

如果需要向评委或协作者展示“整理后的 Skill 包结构”，优先打开：

- `skill/src/storylock-skill-engine/SKILL.md`

配套参考路径：

1. `skill/src/storylock-skill-engine/references/`
2. `skill/src/storylock-skill-engine/assets/migrated/`

## 4. 演示时建议优先聚焦的能力

建议优先展示：

| Skill | 展示理由 |
|-------|---------|
| `LocalPasswordFillSkill` | 输出登录字段，评委易于理解 |
| `ChallengeSigningAuthorizationSkill` | 输出签名结果，能力边界直观 |

这两个最能体现“本地授权内核 + 上层 Skill 包装”的结构。

## 5. 演示时需要避免的误表述

不应把 Demo 描述成：

1. 完整生产系统
2. 全部安全边界本体
3. 已托管全部业务流程

更准确的表述应为：

1. 这是 Skill 层与 Host 层的演示入口
2. Core 仍然是底层正式授权判断的中心
3. Demo 负责展示“能力如何被调用”，而不是替代底层设计文档

## 6. 推荐演示顺序

建议按下面顺序展示：

1. 先展示 `skill/src/storylock-skill-engine/SKILL.md`
   - 说明 StoryLock 已经有一份可复用 Skill 包结构
2. 再展示 `story-lock/examples/04-skill-layer-demo.mjs`
   - 说明 skill-layer 的最小调用闭环
3. 最后展示 `browser-demo.html` 或 `story-workbench.html`
   - 说明这套能力不是概念包装，而是有更完整的宿主验证路径
