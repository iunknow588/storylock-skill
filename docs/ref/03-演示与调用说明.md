# StoryLock 演示与调用说明

## 1. 环境要求

当前代码使用 Node.js `node:sqlite`，建议使用：

```powershell
node -v
```

Node.js 版本应为 22.0.0 或以上。

## 2. 最小自测演示

在 `skill` 目录下分别执行：

```powershell
npm run selftest
```

这四个命令分别验证：

1. 第一层故事处理能力。
2. 第二层对象强度、九宫格、本地授权、SQLite 审计。
3. 第三层远程签名、密码填充和脱敏。
4. 兼容演示包基础能力。

## 3. 三层端到端签名演示

在 `skill/src/storylock-remote-gateway-skill` 下执行：

```powershell
npm run selftest:e2e
```

该演示会串联：

1. 第三层 `requestSignature`。
2. 第二层对象强度策略。
3. 第二层九宫格验证。
4. 第二层本地授权。
5. 本地签名执行器。
6. 第三层脱敏返回。
7. SQLite `audit_log` 写入签名授权审计。

成功输出：

```text
StoryLock three-layer e2e selftest passed.
```

## 4. SQLite 清理命令

在 `skill/src/storylock-local-story-access-skill` 下执行：

```powershell
npm run cleanup -- --development-memory-secret-store
```

如果清理持久化 SQLite 文件，需要显式指定 SecretStore 策略：

```powershell
npm run cleanup -- --db E:\path\to\storylock.db --use-platform-secret-store
```

开发测试可以使用：

```powershell
npm run cleanup -- --db E:\path\to\storylock.db --development-memory-secret-store
```

## 6. 演示时建议重点

建议优先展示：

| 演示 | 原因 |
| --- | --- |
| `npm run selftest:e2e` | 最能说明三层链路已经串起来 |
| 第二层 selftest 输出 JSON | 能证明防重放、锁定、审计、cleanup 均有覆盖 |
| 第三层 selftest | 能证明 `requestSignature`、`requestPasswordFill` 和脱敏 |

## 7. 演示时避免的误表述

不要说：

1. 已经实现完整多链钱包系统。
2. 已经实现多平台发布账号管理。
3. 远程网关可以读取故事或密码。

更准确的说法是：

1. 当前已实现本地授权与远程请求包装的核心链路。
2. 签名和密码填充是两个典型应用入口。
3. 更多应用领域属于后续探索。
