## 4. 提交前检查项

- [ ] Node.js 版本是否为 22 或以上。
- [ ] 根目录 `npm run test` 是否通过。
- [ ] 文档是否统一使用 `requestSignature` 与 `requestPasswordFill`。
- [ ] 是否准确说明当前 Android 侧仍为宿主 mock 验证，而非仓库内完整真机工程。
- [ ] 是否避免把第二层描述成故事读取或写回层。
- [ ] 是否说明当前应用扩展只是探索方向，不是已完成能力。
- [ ] 是否说明远程侧不直接持有密码、私钥或九宫格答案。

## 5. 推荐中文提交口径

建议采用以下表达：

> StoryLock 是一个本地优先的授权访问 Skill 项目。它通过故事处理、本地访问授权、远程网关三层结构，将九宫格验证、短时授权、签名请求和 Web2 密码填充包装为可调用能力，同时把长期秘密和授权判断保留在本地边界内。

## 6. 当前不建议写入提交材料的说法

以下说法容易过度承诺，建议避免：

1. “已经支持多链多钱包生产级签名管理”。
2. “已经支持多平台内容发布账号管理”。
3. “远程网关可以读取或写回故事对象”。
4. “兼容演示包就是完整生产安全边界”。
