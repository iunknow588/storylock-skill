# StoryLock 三包接口契约

## 1. 当前基线

本文档以当前 `skill/src` 源码为准，定义三个主线运行包之间的最小接口契约：

1. `src/skills/local-story-processing`
2. `src/skills/local-story-access`
3. `src/skills/remote-gateway`

历史上出现过的 story read/write、`requestChallengeSign`、旧目录 `src/storylock-*` 等描述，不再作为当前主线契约。

## 2. 包职责

| 包 | 层级 | 当前职责 |
| --- | --- | --- |
| `src/skills/local-story-processing` | 第一层 | 本地故事草稿、润色、题集强度评估 |
| `src/skills/local-story-access` | 第二层 | 对象强度判断、九宫格验证、本地授权、撤销、审计 |
| `src/skills/remote-gateway` | 第三层 | 远程请求包装、字段校验、EIP-712 表达、结果脱敏 |

## 3. 第一层契约

第一层当前暴露：

1. `StoryDraftSkill`
2. `StoryRefineSkill`
3. `StrengthReviewSkill`

第一层只负责本地文本处理和题集质量判断，不创建 challenge，不发放 session，也不直接访问长期敏感对象。

### 3.1 强度评估输入

```json
{
  "questions": [
    {
      "nodeId": "q-001",
      "validAnswers": ["answer"],
      "distractors": ["d1", "d2", "d3", "d4", "d5", "d6", "d7", "d8"]
    }
  ]
}
```

### 3.2 强度评估约束

1. 当前主线按 24 题题集为目标。
2. 每题应能组成 9 个唯一候选项。
3. 输出只描述强度判断、问题统计、建议动作，不返回第二层或第三层执行状态。

## 4. 第二层契约

第二层当前暴露：

1. `ObjectStrengthPolicySkill`
2. `GridChallengeSkill`
3. `LocalAuthorizationSkill`
4. `LocalRevocationSkill`

第二层当前持久化的 SQLite 状态包括：

1. `challenge_state`
2. `session_store`
3. `request_store`
4. `nonce_store`
5. `failure_window`
6. `answer_digest_set`
7. `audit_log`

### 4.1 对象强度判断

输入示例：

```json
{
  "identityId": "identity-001",
  "objectRef": "wallet-key-main",
  "objectType": "signature_key",
  "requestedAction": "signature"
}
```

输出核心字段：

```json
{
  "requiredStrength": "high",
  "gridPolicy": {
    "gridSize": 9,
    "requiredCells": 9
  }
}
```

### 4.2 九宫格验证生成

调用方必须提供：

1. `identityId`
2. `objectRef`
3. `requiredStrength`
4. `requestId`
5. `nonce`
6. `expiry`

成功返回：

1. `verificationId`
2. `requiredStrength`
3. `grid.cells`
4. `expiresAt`

约束：

1. `grid.cells` 不返回答案明文。
2. 第二层必须校验重放与过期。
3. 默认生产口径不启用 legacy fallback。

### 4.3 本地授权

输入示例：

```json
{
  "identityId": "identity-001",
  "objectRef": "wallet-key-main",
  "verificationId": "chl-001",
  "allowedAction": "signature",
  "answers": [
    {
      "cellId": "cell-1",
      "answer": "normalized answer"
    }
  ]
}
```

成功返回至少包含：

1. `approved`
2. `authorizationId`
3. `identityId`
4. `objectRef`
5. `allowedAction`
6. `expiresAt`

预算策略：

1. `signature` 与 `password_fill` 默认 `readBudget=1`、`writeBudget=0`
2. 普通 `authorize` 默认 `readBudget=0`、`writeBudget=0`
3. session 是短时授权结果，不是长期能力令牌

### 4.4 题集导入契约

第二层当前题集导入入口：

1. `npm run generate:question-set-template`
2. `npm run validate:question-set`
3. `npm run import:question-set`

对应 schema：

`src/skills/local-story-access/assets/schemas/question-set-master.schema.json`

## 5. 第三层契约

第三层当前主线方法只有两个：

1. `requestSignature`
2. `requestPasswordFill`

### 5.1 requestSignature

业务输入示例：

```json
{
  "requestId": "req-sign-001",
  "nonce": "nonce-001",
  "expiry": 1760000300,
  "identityId": "identity-001",
  "keyId": "wallet-key",
  "algorithm": "ed25519",
  "payload": "sign this payload",
  "resourceId": "wallet-main",
  "primaryRole": "private_key",
  "eip712Nonce": "1001"
}
```

第三层会构造 `StoryLockSignatureRequest` 风格的 EIP-712 payload，并通过以下任一方式执行：

1. `transport`
2. `signatureExecutor`

当未显式传入 domain 时，第三层会依据环境变量解析 `demo`、`test` 或 `production` 的 EIP-712 domain。

### 5.2 requestPasswordFill

业务输入示例：

```json
{
  "requestId": "req-fill-001",
  "nonce": "nonce-002",
  "expiry": 1760000300,
  "identityId": "identity-001",
  "credentialRef": "cred-example",
  "targetOrigin": "https://example.com"
}
```

第三层默认附带：

1. `capability=requestPasswordFill`
2. `scope=password_fill_basic`
3. `requestedRetention=audit_meta_only`
4. `policyHints.noRemoteSecretReturn=true`

## 6. 主线 Schema

第三层主线 schema：

1. `src/skills/remote-gateway/assets/schemas/remote-gateway-request.schema.json`
2. `src/skills/remote-gateway/assets/schemas/remote-gateway-response.schema.json`

请求 schema 约束：

1. `requestId`
2. `capability`
3. `scope`
4. `payload`
5. `nonce`
6. `expiry`

响应 schema 约束：

1. `status`
2. `capability`
3. `executionLocation`
4. `result`
5. `error`

## 7. 错误码

当前主线错误码前缀为 `SLG-*`：

| 错误码 | 类型 | 含义 |
| --- | --- | --- |
| `SLG-001` | `validation_error` | 请求字段不合法 |
| `SLG-003` | `authorization_failed` / `challenge_failed` | 本地授权未通过 |
| `SLG-008` | `redaction_required` | 返回结果必须脱敏 |
| `SLG-010` | `question_set_unavailable` / `secret_unavailable` | 题集或受保护材料不可用 |
| `SLG-011` | `request_expired` | 请求过期 |
| `SLG-013` | `replay_detected` | requestId 或 nonce 冲突 |

## 8. 禁止规则

1. 第三层不得把第二层内部方法当作远程公开接口。
2. 第三层不得返回密码、私钥、九宫格答案、`signingKeyBytes` 等敏感明文。
3. 第二层不得重新承担 story read/write 主线职责。
4. 第一层不得绕过第二层直接访问受保护对象。
5. `src/engine` 是兼容与演示包，不是新的第四层安全边界。
