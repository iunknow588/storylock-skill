### 3.2 Skill定位与边界.md

**建议增加"代码映射"章节**：

```markdown
## 代码映射示例

### Skill 层代码特征

```javascript
// Skill 层：只负责输入输出契约和自描述
class StoryDraftAssistSkill {
  skillId() { return 'story-draft-assist'; }
  
  async run(input) {
    // 1. 验证输入（不访问秘密）
    ensureNonEmptyString(input.objective);
    
    // 2. 调用 Host 层接口（不直接访问存储）
    const context = await this.host.getStoryContext(input.identityId);
    
    // 3. 返回结构化结果
    return { draft: generatedDraft, metadata: {...} };
  }
}
```

### Host 层代码特征

```javascript
// Host 层：负责敏感执行边界
class StoryLockHost {
  async createChallenge(identityId, scope) {
    // 直接访问加密存储，创建挑战
    return challenge;
  }
  
  async readSecretObject(identityId, sessionId, secretObjectId) {
    // 验证会话有效性后读取秘密
    return secret;
  }
}
```

### 反模式：Skill 层越界访问

```javascript
// ❌ 错误：Skill 直接访问存储
class BadSkill {
  async run() {
    const secret = await fs.readFile('./encrypted-vault/secret.key'); // 越界！
  }
}
```
```

**理由**：帮助开发者理解"边界"不是抽象概念，而是代码层面的约束。

### 3.5 本地Agent网关设计.md

**现状**：网关接口定义清晰，但**缺少错误处理规范**。

**建议增加"错误码规范"**：

```markdown
## 错误码规范

### 错误码结构

```json
{
  "errorCode": "SLG-001",
  "errorType": "CAPABILITY_NOT_AVAILABLE",
  "message": "请求的本地能力当前不可用",
  "suggestedAction": "请检查本地Agent是否已启动，或尝试其他能力",
  "retryable": true
}
```

### 错误码列表

| 错误码 | 错误类型 | 说明 | 可重试 | 处理建议 |
|--------|---------|------|--------|---------|
| SLG-001 | CAPABILITY_NOT_AVAILABLE | 能力不可用 | 是 | 检查本地Agent状态 |
| SLG-002 | CHALLENGE_REQUIRED | 需要挑战验证 | 是 | 引导用户完成挑战 |
| SLG-003 | CHALLENGE_FAILED | 挑战验证失败 | 是 | 重新提交答案 |
| SLG-004 | SESSION_EXPIRED | 会话已过期 | 否 | 重新创建会话 |
| SLG-005 | SCOPE_INSUFFICIENT | 权限不足 | 否 | 请求更高权限 |
| SLG-006 | SECRET_NOT_FOUND | 秘密对象不存在 | 否 | 检查对象ID |
| SLG-007 | HOST_NOT_IMPLEMENTED | Host接口未实现 | 否 | 检查宿主实现 |
| SLG-008 | REDACTION_REQUIRED | 需要脱敏 | 否 | 对结果进行脱敏处理 |
```

**理由**：远程Agent需要明确的错误处理指引，避免无限重试或错误传播。

**现状**：协议参考全面，但**缺少与 StoryLock 具体结合的代码示例**。

## EIP-712 适配示例

### StoryLock 签名请求结构

```javascript
const domain = {
  name: 'StoryLock',
  version: '1',
  chainId: 1, // 或特定链
  verifyingContract: '0x...' // 如适用
};

const types = {
  StoryLockSignatureRequest: [
    { name: 'action', type: 'string' },
    { name: 'resource', type: 'string' },
    { name: 'scope', type: 'string' },
    { name: 'expiry', type: 'uint256' },
    { name: 'nonce', type: 'uint256' },
    { name: 'requestedBy', type: 'address' },
    { name: 'delegationContext', type: 'string' }
  ]
};

const value = {
  action: 'password_fill',
  resource: 'site:example.com',
  scope: 'vault_read_basic',
  expiry: Math.floor(Date.now() / 1000) + 3600, // 1小时后过期
  nonce: Date.now(),
  requestedBy: '0xRemoteAgentAddress',
  delegationContext: 'user:12345/agent:67890'
};
```

### 本地签名流程

```javascript
// 1. 远程Agent构造EIP-712请求
const signatureRequest = createEIP712Request(domain, types, value);

// 2. 通过网关发送给本地Agent
const response = await localAgentGateway.requestChallengeSign({
  requestId: 'req-123',
  capability: 'challenge_sign',
  scope: 'vault_read_basic',
  payload: signatureRequest,
  policyHints: { minLevel: 'L3' }
});

// 3. 本地Agent验证并签名
if (response.status === 'success') {
  const signature = response.result.signature;
  const signerAddress = response.result.signerAddress;
  
  // 4. 验证签名（使用EIP-1271兼容方式）
  const isValid = await verifyEIP712Signature(domain, types, value, signature, signerAddress);
}
```
```

**理由**：从协议参考到代码实现，降低开发者的适配成本。

### 4.1 快速决策指南.md（新增）

**目的**：一页纸判断"这个能力该放哪层？"

```markdown
# StoryLock 快速决策指南

## 决策流程图

```
开始
  │
  ▼
这个能力需要访问以下任一内容吗？
  - challenge answers
  - session material
  - secret-object reads
  - signing-key access
  │
  ├── 是 → 本地专属（第一层或第二层）
  │
  └── 否 → 这个能力只处理已脱敏文本或非秘密结构化数据吗？
            │
            ├── 是 → 远程安全（第三层）
            │
            └── 否 → 需要进一步分析（参考完整文档）
```

## 常见场景对照表

| 场景 | 能力示例 | 推荐层级 | 关键判断依据 |
|------|---------|---------|------------|
| 用户说"帮我写个故事草稿" | StoryDraftAssistSkill | 第一层（本地） | 可能涉及私密记忆 |
| 用户说"润色这个故事"（已脱敏） | StoryRefineAssistSkill | 第三层（远程） | 输入已脱敏 |
| 用户说"我要登录" | LocalPasswordFillSkill | 第一层（本地） | 需要读取密码 |
| 用户说"帮我签名这个交易" | ChallengeSigningAuthorizationSkill | 第一层（本地） | 需要签名密钥 |
| 第三方Skill说"我需要签名" | 代理签名请求 | 第三层（远程）→ 第一层（本地） | 远程请求，本地执行 |
| 评估题集强度 | StrengthReviewSkill | 视内容而定 | 题集是否敏感 |

## 反模式清单

### ❌ 不要把以下能力放在远程

1. **直接读取本地加密存储**
   - 错误：远程Skill直接访问文件系统
   - 正确：通过本地Agent网关请求，本地Host执行读取

2. **长期持有私钥或签名能力**
   - 错误：远程Agent保存私钥副本
   - 正确：每次签名都通过本地授权流程

3. **缓存challenge answers**
   - 错误：远程Agent保存用户答案
   - 正确：答案只在本地验证，不传输

4. **绕过Host授权直接访问Secret**
   - 错误：Skill直接调用storage层
   - 正确：Skill通过Host接口，Host执行授权检查

### ❌ 不要把以下能力放在本地（不必要）

1. **通用文本处理（已脱敏）**
   - 错误：在本地运行大型语言模型做通用润色
   - 正确：脱敏后发送到远程服务，结果返回本地

2. **非敏感数据分析**
   - 错误：在本地执行大量计算分析公开数据
   - 正确：远程执行，只返回结果
```

### 4.2 术语表.md（新增）

**目的**：统一术语，避免歧义

```markdown
# StoryLock 术语表

| 术语 | 英文 | 定义 | 使用场景 |
|------|------|------|---------|
| 挑战 | Challenge | 用户需要回答的问题集，用于身份验证 | "创建挑战"、"提交挑战答案" |
| 会话 | Session | 挑战验证通过后建立的临时访问权限 | "会话有效期"、"会话过期" |
| 秘密对象 | Secret Object | 加密存储的敏感数据（密码、密钥等） | "读取秘密对象"、"秘密对象ID" |
| 宿主 | Host | 提供本地执行环境的运行时 | "Host层"、"Host接口" |
| 技能 | Skill | 具有明确输入输出契约的能力单元 | "主Skill"、"内部Skill" |
| 网关 | Gateway | 远程Agent访问本地能力的受控接口 | "本地Agent网关" |
| 委托签名 | Delegated Signing | 第三方请求本地执行签名的机制 | "代理签名请求" |
| 脱敏 | Redaction | 去除个人标识信息，使数据可安全远程传输 | "脱敏输出"、"脱敏级别" |
| 题集 | Question Set | 用于挑战的24个问题集合 | "24题题集"、"题集强度" |
| 访问强度 | Access Level | 访问秘密所需的挑战正确数（L1-L5） | "L3级别访问" |
```
