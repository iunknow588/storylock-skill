本次检查以 `docs/design/cn`、`docs/design/en`、`docs/ref`、`docs/test`、`docs/management/back` 与 `src`、`android-host` 为依据。

当前主线已经超过“最小可演示闭环”阶段，现状包括：

1. 真实题集注册、题集状态与版本字段已落地。
2. 主动撤销、生产 EIP-712 domain 校验、SecretStore 生产约束已落地。
3. 文档一致性检查、自测、题集导入 dry-run 已可运行。

当前工作的重点是把这些能力继续固化成可维护、可部署、可答辩、可迁移的结构。

## 2. 移动端本地宿主与第三层远程网关

### 2.1 当前判断

“移动端本地宿主 + 第三层远程网关”的结构是合适的。

建议采用：

1. Android 本地宿主承载第一层与第二层。
2. 第三层作为远程网关层，通过云端 HTTP 入口承接外部请求。
3. 链上能力如果接入，应作为审计锚点或配置证明层，而不是本地授权主路径。

### 2.2 已有落地

1. `api/storylock-gateway.mjs`
2. `src/storylock-remote-gateway-skill/vercel-handler.js`
3. `src/storylock-remote-gateway-skill/scripts/selftest-vercel-android.mjs`
4. `src/storylock-remote-gateway-skill/references/android-host.md`
5. `docs/ref/05-Android宿主实现规范.md`

1. 题集主档导入命令已存在，但生产发布、弃用、迁移、回滚流程还需继续固化。

1. Windows/Linux SecretStore 约束已经明确，但跨平台检查记录模板还可继续补充。

### 3.4 Android 真宿主

2. `GET /health` 与 `POST /execute` 已接入最小运行时，但 challenge 和 session 仍是当前阶段的最小实现。
3. 设备登记、`deviceId` / `appInstanceId`、共享密钥升级与宿主撤销流程尚未实现。
4. 真机网络可达性方案、宿主暴露方式与部署说明尚未实现。

### 3.5 链上接入
