# 易安与 StoryLock 使用说明

易安用于把需要确认的重要请求交回到用户自己的本地设备上处理。用户通过易安远程访问接口下载并安装易安 App，完成一次绑定后，就可以在本地设备上查看请求来源、理解请求内容，并确认是否继续。

当前线上入口：

`https://yian.cdao.online`

## 先看整体

易安是 StoryLock 使用流程中的云端入口与本地确认入口。它负责把外部请求带到你的本地设备，并让你在本地确认后再继续。

从软件结构层次上，整体分为三部分：

1. 云端入口：易安远程访问接口运行在云服务平台上，负责展示说明、提供下载、发起绑定、接收外部 Agent 或服务的请求，并展示请求状态。
2. 本地助理：你下载的易安 App 安装在自己的本地设备上，承载私人智能助理。它负责接收请求、解释来源和风险，并把需要确认的内容交给本地核心处理。
3. 本地核心：StoryLock 本地核心负责故事文件、密钥隔离和敏感确认。它只在本地设备侧工作，把最小必要结果返回给私人智能助理。

所以，你下载的易安 App 不是远程云服务本身，而是连接“易安云端入口”和“StoryLock 本地核心”的本地确认入口。平时你主要使用易安远程访问接口下载安装和绑定；当远端发起密码填充、签名或其他交互授权请求时，需要在自己的本地设备上查看请求状态并完成确认。

## 你会用到什么

1. 易安远程访问接口：用于下载 App、打开绑定入口、查看请求状态和安装包信息。
2. 易安 App：安装在你的本地设备上，用于接收绑定信息和确认请求。
3. 私人智能助理：运行在你的本地设备上，帮助解释请求来源、内容和风险，也可以辅助生成故事模板草稿。
4. StoryLock 本地核心：负责本地确认、故事存储和敏感执行。

这里说的“本地设备”可以是手机，也可以是其他由你自己控制的设备。

## 快速开始

1. 打开易安远程访问接口：`https://yian.cdao.online`。
2. 点击“下载易安 App”，下载当前安装包。
3. 安装前核对页面展示的版本号、文件大小和校验值。
4. 在本地设备上安装易安 App。
5. 回到易安远程访问接口，点击“打开绑定入口”。
6. 按本地设备上的提示完成首次绑定。
7. 回到易安远程访问接口的“请求状态”区域，确认本地设备是否可以接收请求。

## 如何确认请求

绑定完成后，当有需要本地确认的请求出现时：

1. 先在本地设备上查看请求来源。
2. 阅读私人智能助理给出的内容说明和风险提示。
3. 确认请求与你正在进行的操作一致。
4. 按本地设备提示完成解锁、生物识别或设备凭据确认。
5. 确认完成后，结果会返回给私人智能助理，再由私人智能助理按需要同步确认状态。

如果你不认识请求来源，或请求内容与你的操作不一致，请不要确认。

## 查看请求状态

在易安远程访问接口中打开“请求状态”区域，可以查看：

1. 是否有正在等待处理的请求。
2. 请求来自哪里，需要执行什么操作。
3. 请求处于待确认、已确认、已取消或已完成状态。
4. 本地设备是否可以接收新的请求。
5. 当前安装包版本、大小和校验值。

如果本地设备暂时无法接收请求，可以先打开易安 App，检查本地设备网络，再回到易安远程访问接口重新查看。仍然无法恢复时，可以重新打开绑定入口完成一次新的绑定。

## 使用注意事项

1. 只从你确认可信的易安远程访问接口下载安装包。
2. 不要从陌生链接、聊天转发或不明来源下载安装包。
3. 安装前核对版本号、文件大小和 SHA-256。
4. 不要把绑定链接转发给他人。
5. 保持本地设备网络可用，方便私人智能助理接收请求和返回确认状态。
6. 遇到不认识的请求，先暂时不确认，再检查来源。

## 常见问题

### 请求状态没有更新怎么办？

先确认本地设备网络可用，然后打开易安 App 等待几秒，再回到易安远程访问接口查看请求状态。如果仍然没有更新，可以重新绑定。

### 绑定链接应该连接到什么设备？

绑定链接用于连接当前使用环境和一个可信的本地设备。重点不是设备属于谁，而是你是否确认这台设备可信、可控制，并且准备让它接收和处理后续请求。

### 安装时系统提示风险怎么办？

只安装来自可信易安远程访问接口的安装包。安装前核对版本号、文件大小和 SHA-256；如果来源不确定，先不要安装。

### 换了本地设备怎么办？

先从旧设备导出或备份原来的本地文件，再把这些文件上传或迁移到新设备。确认文件已经在新设备上可用后，重新下载安装易安 App，回到易安远程访问接口完成绑定，并查看请求状态。旧设备不再使用时，请停止使用旧连接。
