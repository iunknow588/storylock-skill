1. 能力划分
2. 边界判定

## 当前阶段技术栈建议

当前阶段建议固定为：

1. 运行时：Node.js / TypeScript（建议 Node.js >= 18 LTS）
2. 本地存储：SQLite 单文件数据库
3. 本地敏感存储：操作系统密钥链或平台安全存储
4. 第三层远程包装：后续再决定是否完整接入 Pharos Skill Engine

## 当前阶段目标

当前阶段优先目标不是把所有远程机制做完，而是：

1. 固定三部分 Skill 边界
2. 固定三个 Skill 包边界
3. 先让本地 Agent 可实现
4. 再处理第三层远程接入

## 当前阶段不做的事

为了避免过早分散实现精力，当前阶段明确暂不追求：

1. 完整链上验证
2. EIP-1271 实际落地
3. ERC-4337 执行层集成
4. 完整第三层云端部署
