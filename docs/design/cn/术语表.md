# StoryLock 术语表

## 目的

统一设计文档中的核心术语，避免同一概念反复换说法。

| 术语 | 定义 |
| --- | --- |
| Skill | 具有明确输入输出契约的能力单元 |
| 主 Skill | 直接面向用户任务或评审演示的 Skill |
| 内部 Skill | 作为底层构件存在的 Skill |
| 编排示例 | 说明如何串联多个 Skill 的示例对象或示例流程 |
| Host | 提供本地敏感执行边界的运行时 |
| Gateway | 远程 Agent 访问本地能力的受控入口 |
| Challenge | 用于授权验证的问题或挑战流程 |
| Session | challenge 通过后建立的短时访问状态 |
| Secret Object | 被 StoryLock 保护的秘密对象 |
| 故事处理 Skill | 负责编辑、修改、完善故事内容的本地 Skill |
| 本地访问授权 Skill | 负责对象强度判断、九宫格验证、短时授权和审计的本地 Skill |
| 远程网关 Skill | 面向第三方请求、包装、编排与委托的远程能力 |
| 代理签名 | 第三方请求本地完成签名，而不是自己持有私钥 |
| 易安远程入口 | 对外提供下载、绑定、状态查看和请求接入的云端入口 |
| 私人智能助理 | 运行在用户可控本地设备上的解释、确认与转接层 |
| StoryLock 本地核心 | 负责离线敏感执行、授权判断和本地秘密管理的核心边界 |
| Android 宿主 | 当前用于承载本地能力的 Android 侧宿主接口模型或平台实现方向 |
| Windows 宿主 | Windows 平台侧的本地宿主实现方向 |
| active question set | 当前启用并可用于 challenge 的题集集合 |
| legacy fallback | 兼容演示或开发场景下的旧回退路径，生产默认不启用 |
| redaction | 脱敏，指跨边界返回前对敏感信息做删除、遮挡、摘要化或最小化暴露 |
