# StoryLock 平台密钥存储适配指南

| 项目 | 内容 |
| --- | --- |
| 文档版本 | v1.0 |
| 日期 | 2026-06-16 |
| 状态 | 当前阶段采用 |
| 适用范围 | 本地 Host / 第二层 / 第三层本地网关 |

## 目的

本文档用于明确不同平台下长期敏感材料应如何保存。

## 保护对象

当前阶段建议放入平台安全存储的对象：

1. `masterSalt`
2. 本地签名根密钥
3. 恢复材料包装密钥
4. 平台级凭据别名与句柄

## Windows

推荐优先级：

1. 系统保护存储能力
2. 凭据管理能力

## macOS

当前阶段建议：

1. 优先使用 Keychain
2. 以服务名 + 账户名方式保存密钥别名
3. 只把真正长期材料放入 Keychain，运行态 session 不进入 Keychain

## Linux

当前阶段建议：

1. 优先使用 Secret Service / libsecret 兼容能力
2. 若目标环境不存在桌面密钥环，则必须显式降级提示
3. 不建议静默降级为明文文件

## 统一适配接口建议

建议在代码中抽象出统一接口：

```ts
interface SecretStore {
  getSecret(key: string): Promise<Uint8Array | null>;
  setSecret(key: string, value: Uint8Array): Promise<void>;
  deleteSecret(key: string): Promise<void>;
  listKeys?(prefix: string): Promise<string[]>;
}
```

## 命名建议

建议统一键命名：

1. `storylock/masterSalt`
2. `storylock/signingRoot/<identityId>`
3. `storylock/recoveryKey/<identityId>`

## 不应放入平台密钥存储的对象

以下对象当前阶段不建议放入平台密钥存储：

1. `requestId` 去重记录
2. `nonce` 去重记录
3. `session` 运行态记录
4. 大体积题集密文主体

## 降级原则

若平台安全存储不可用：

1. 默认返回明确错误
2. 由 Host 决定是否允许进入“开发模式”
3. 开发模式必须有显式风险提示

## 开发模式安全边界

当前阶段若启用开发模式，建议必须同时满足：

1. 仅限本地开发环境
2. 由用户或开发者显式确认
3. 启动时输出清晰告警
4. 审计日志中记录 `developmentMode = true`

开发模式下仍应遵守：

1. 不允许默认静默写入明文长期私钥
2. 不允许去掉 challenge / session / scope 校验
3. 不允许把本地敏感结果直接暴露给第三层

建议风险提示至少包含：

1. 当前不使用正式平台密钥存储
2. 当前环境只适合本地开发或测试
3. 不可视为生产安全配置

## 当前阶段结论

当前阶段实现时，只需要做到两件事：

1. 长期敏感材料走统一 `SecretStore`
2. 普通运行态数据走 SQLite
