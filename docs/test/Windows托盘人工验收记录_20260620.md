# Windows 托盘人工验收记录

| 项目 | 内容 |
| --- | --- |
| 文档版本 | v1.0 |
| 日期 | 2026-06-20 |
| 适用范围 | Windows 本地宿主 `ui-tray` 首版 |
| 目的 | 记录脚本无法替代的桌面托盘可见性和菜单点击行为 |

## 1. 验收入口

```powershell
scripts\windows\start_windows_host_tray_manual_check.cmd
```

可选指定端口：

```powershell
scripts\windows\start_windows_host_tray_manual_check.cmd -Port 4512
```

## 2. 前置自动检查

```powershell
npm run test:windows-host
npm run test:windows-host-features
npm run test:windows-tray-readiness
```

## 3. 人工检查项

| 编号 | 检查项 | 通过标准 | 结果 | 记录 |
| --- | --- | --- | --- | --- |
| TRAY-01 | 托盘图标可见 | 启动后 Windows 通知区域出现 Yian / StoryLock 托盘图标 | 待验收 |  |
| TRAY-02 | 打开本地管理页 | 菜单项能打开 `http://127.0.0.1:<port>/ui`，页面显示本地宿主状态 | 待验收 |  |
| TRAY-03 | 查看健康状态 | 菜单项能打开 `http://127.0.0.1:<port>/health`，返回健康 JSON | 待验收 |  |
| TRAY-04 | 复制诊断信息 | 菜单项能把脱敏诊断 JSON 复制到剪贴板，内容不含题库答案、私钥、密码或 shared secret | 待验收 |  |
| TRAY-05 | 退出 | 菜单项能停止 host，端口释放，再访问 `/health` 失败 | 待验收 |  |

## 4. 当前结论

当前仅可表述为：`ui-tray` 首版已实现并通过编译与本地 API 自动化检查。

在 TRAY-01 到 TRAY-05 全部记录为通过前，不应表述为：Windows 托盘桌面体验已完成人工验收。
