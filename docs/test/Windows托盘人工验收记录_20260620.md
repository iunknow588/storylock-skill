| 目的 | 记录脚本无法替代的桌面托盘可见性和菜单点击行为 |

## 1. 验收入口

```powershell
scripts\windows\start_windows_host_tray_manual_check.cmd
```

可选指定端口：

```powershell
scripts\windows\start_windows_host_tray_manual_check.cmd -Port 4512
```

可选指定本地预检报告：

```powershell
scripts\windows\start_windows_host_tray_manual_check.ps1 -ReportPath docs\test\Windows托盘人工验收记录_20260620.local.md
```

仅生成预检报告、不启动托盘 UI：

```powershell
scripts\windows\start_windows_host_tray_manual_check.ps1 -PrecheckOnly
```

## 2. 前置自动检查

```powershell
npm run test:windows-host
npm run test:windows-host-features
npm run test:windows-tray-readiness
```

2026-06-22 自动化复核：`npm run test:platform-readiness` 已包含 Windows host loop、Windows tray readiness 和 Windows host feature 编译检查，并已通过。当前仅说明本地 API、`ui-tray`/`ui-slint` feature 编译和托盘手工入口可用；不等同于托盘图标和菜单点击已经人工验收。

2026-06-22 脚本补充：托盘人工启动脚本已支持写入本地预检报告，记录 cargo 路径、端口、数据目录和 TRAY-01 到 TRAY-05 待填项。该报告默认写入 `.temp\windows-tray-manual-report.local.md`；`-PrecheckOnly` 可只生成报告而不启动 UI。该报告不替代本文件中的人工结果。
