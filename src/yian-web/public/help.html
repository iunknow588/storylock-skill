易安与 StoryLock 使用说明

易安用于把需要确认的重要请求交回到用户自己的本地设备上处理。用户通过易安远程访问接口下载并安装易安 App，完成一次绑定后，就可以在本地设备上查看请求来源、理解请求内容，并确认是否继续。

当前线上入口：https://yian.cdao.online

先看整体

从软件结构层次上，整体分为三部分

云端入口

易安远程访问接口运行在云服务平台上，负责展示说明、提供下载、发起绑定、接收外部 Agent 或服务的请求，并展示请求状态。

本地助理

易安 App 安装在自己的本地设备上，承载私人智能助理。它负责接收请求、解释来源和风险，并把需要确认的内容交给本地核心处理。

本地核心

StoryLock 本地核心负责故事文件、密钥隔离和敏感确认。它只在本地设备侧工作，把最小必要结果返回给私人智能助理。

你下载的易安 App 是连接“易安云端入口”和“StoryLock 本地核心”的本地确认入口。当远端发起密码填充、签名或其他交互授权请求时，需要在自己的本地设备上查看请求状态并完成确认。

快速开始

从安装到绑定

  1. 打开易安远程访问接口：https://yian.cdao.online。
  2. 点击“下载易安 App”，下载当前安装包。
  3. 安装前核对页面展示的版本号、文件大小和校验值。
  4. 在本地设备上安装易安 App。
  5. 回到易安远程访问接口，点击“打开绑定入口”。
  6. 按本地设备上的提示完成首次绑定。
  7. 回到“请求状态”区域，确认本地设备是否可以接收请求。

如何确认请求

  1. 先在本地设备上查看请求来源。
  2. 阅读私人智能助理给出的内容说明和风险提示。
  3. 确认请求与你正在进行的操作一致。
  4. 按本地设备提示完成解锁、生物识别或设备凭据确认。
  5. 结果返回给私人智能助理，再按需要同步确认状态。

查看请求状态

  1. 是否有正在等待处理的请求。
  2. 请求来自哪里，需要执行什么操作。
  3. 请求处于待确认、已确认、已取消或已完成状态。
  4. 本地设备是否可以接收新的请求。
  5. 当前安装包版本、大小和校验值。

注意事项

使用前请确认

  1. 只从你确认可信的易安远程访问接口下载安装包。
  2. 不要从陌生链接、聊天转发或不明来源下载安装包。
  3. 安装前核对版本号、文件大小和 SHA-256。
  4. 绑定链接用于连接当前使用环境和一个可信的本地设备。
  5. 保持本地设备网络可用，方便私人智能助理接收请求和返回确认状态。
  6. 遇到不认识的请求，先暂时不确认，再检查来源。
