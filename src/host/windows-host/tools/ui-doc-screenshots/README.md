# UI Doc Screenshots

This directory owns the Windows Host side of the UI documentation screenshot workflow. It is an internal documentation tooling area and is not part of the Windows Host runtime path.

It does not merely call `tools/render-storylock-ui-docs`. This directory coordinates the documentation workflow and also contains the separate live-window capture helper. The Rust renderer owns repeatable static screenshots; the PowerShell helper owns screenshots from a launched executable.

## Layout

1. Static Slint renderer:
   - `tools/render-storylock-ui-docs`
   - usage docs: `tools/render-storylock-ui-docs/docs`
2. Live window capture helper:
   - `src/host/windows-host/tools/ui-doc-screenshots/capture_storylock_ui.ps1`
3. Backward-compatible legacy entry:
   - `docs/management/流程图/tools/capture_storylock_ui.ps1`
4. Documentation output directory:
   - `docs/management/流程图/ui-screenshots`

The legacy PowerShell file is only a forwarding wrapper. New work should update the script in this directory.

## Screenshot Sets

The documentation currently uses these groups:

1. `01-07`: real Windows Host / StoryLock UI screenshots generated by the static Slint renderer.
2. `01-05`: optional live executable captures generated by `capture_storylock_ui.ps1`; these are useful when checking the actual launched windows.
3. `08-26`: supplemental storyboard images for package structure, path continuity, save/apply boundaries, and video/manual narration.

The static renderer is the preferred source for repeatable documentation images. The live capture helper is for spot checks or when the actual executable behavior is under review.

## Prerequisites

Run these commands from a Windows PowerShell session with Rust available in `PATH`. The examples below assume the current directory is the `skill` directory.

The static renderer requires the local Slint sources under `src/host/windows-host/src/slint_ui`. The live capture helper additionally requires a built Windows Host executable at:

```text
src/host/windows-host/target/codex-build/debug/yian-windows-host.exe
```

You can pass `-ExePath` to use another build output.

## Static Renderer

Generate the repeatable Slint-based documentation screenshots:

```powershell
cargo run --manifest-path .\tools\render-storylock-ui-docs\Cargo.toml -- .\docs\management\流程图\ui-screenshots
```

If the output path is omitted, the renderer writes to the same `docs/management/流程图/ui-screenshots` directory by default.

Expected generated files:

```text
01_host_remote_web.png
02_host_storylock_page.png
03_host_settings.png
04_storylock_core_empty_mode.png
05_storylock_authorization_challenge.png
06_storylock_core_unlocked_questions.png
07_storylock_core_save_current_package.png
```

## Live Window Capture

Capture the running executable flow:

```powershell
.\src\host\windows-host\tools\ui-doc-screenshots\capture_storylock_ui.ps1
```

Use explicit paths when needed:

```powershell
.\src\host\windows-host\tools\ui-doc-screenshots\capture_storylock_ui.ps1 `
  -ExePath .\src\host\windows-host\target\codex-build\debug\yian-windows-host.exe `
  -OutputDir .\docs\management\流程图\ui-screenshots `
  -TimeoutSeconds 30
```

The helper starts the executable if needed, finds visible windows by title, clicks through the StoryLock flow, and captures:

```text
01_host_remote_web.png
02_host_storylock_page.png
03_host_settings.png
04_storylock_core_empty_mode.png
05_storylock_authorization_challenge.png
```

Because it interacts with real windows, do not move the mouse or cover the target windows while it is running.

## Validation

Check that the renderer still builds:

```powershell
cargo check --manifest-path .\tools\render-storylock-ui-docs\Cargo.toml
```

Check that the expected screenshot files exist:

```powershell
Get-ChildItem .\docs\management\流程图\ui-screenshots |
  Sort-Object Name |
  Select-Object Name, Length, LastWriteTime
```

For script syntax only:

```powershell
$null = [System.Management.Automation.Language.Parser]::ParseFile(
  ".\src\host\windows-host\tools\ui-doc-screenshots\capture_storylock_ui.ps1",
  [ref]$null,
  [ref]$null
)
```

## Maintenance Rules

1. Keep reusable screenshot tooling in this directory.
2. Keep generated images in `docs/management/流程图/ui-screenshots`.
3. Keep the Rust static renderer in `tools/render-storylock-ui-docs` unless the repository layout changes again.
4. Update this README whenever filenames, click coordinates, window titles, or output conventions change.
5. Keep the legacy docs wrapper while older documentation or notes still reference it.
