# 概述

StoryLock 九宫格插件负责“展示题目、收集用户选择”。它不是判卷模块，也不是授权模块。

## 插件定位

输入：

- 草稿 JSON
- 需要展示的题目数量 `required_cells`

处理：

- 读取题目标识、题目文本和选项文本
- 每题生成固定 9 个选项格，不足补空字符串
- 根据用户点击切换选择状态

输出：

- `StoryLockChallengeCell[]`：用于渲染题目
- `Vec<Vec<String>>`：每题对应的用户选择

## 职责边界

插件内：

- `StoryLockChallengeCell`
- `create_open_challenge_from_draft`
- `toggle_selection`
- `normalize_answer`

插件外：

- 正确答案存储
- `isCorrect` 或其他答案标记解析
- 用户答案验证
- 解锁授权
- UI 框架适配和窗口生命周期

## 为什么不传入正确答案

九宫格只是一个输入 UI。把正确答案传入展示组件会让展示层依赖验证层，降低复用性，也会在客户端场景中增加答案泄露风险。

因此，本插件不提供任何需要正确答案的接口。
