# 接口说明

本插件是独立的九宫格展示和输入收集 UI 核心。它只接收题目和选项，只输出用户选择，不持有、不解析、不接收正确答案。

## `StoryLockChallengeCell`

```rust
pub struct StoryLockChallengeCell {
    pub cell_id: String,
    pub prompt_text: String,
    pub answer_options: Vec<String>,
}
```

字段职责：

| 字段 | 说明 |
| --- | --- |
| `cell_id` | 题目标识，提交用户答案时用于外部验证 |
| `prompt_text` | 题目文本 |
| `answer_options` | 九宫格选项，固定 9 个，不足以空字符串补齐 |

结构体中不包含 `expected_answers`、`expected_count` 或任何正确答案标记。

## `create_open_challenge_from_draft`

```rust
pub fn create_open_challenge_from_draft(
    draft: &serde_json::Value,
    required_cells: usize,
) -> anyhow::Result<Vec<StoryLockChallengeCell>>
```

从草稿 JSON 中提取指定数量的题目展示数据。

读取字段：

- `nodes`
- `nodes[].nodeId`
- `nodes[].question`
- `nodes[].answerOptionsLocalOnly[].text`

不会读取 `isCorrect`。即使草稿中存在该字段，也会被九宫格插件忽略。

错误情况：

- `nodes` 不存在或不是数组
- 题目数量少于 `required_cells`
- 题目文本为空
- 一个题目的所有选项文本都为空

## `toggle_selection`

```rust
pub fn toggle_selection(
    cells: &[StoryLockChallengeCell],
    selections: &mut [Vec<String>],
    current_index: usize,
    answer_index: usize,
)
```

切换某个选项的用户选择状态。

- 空字符串选项会被忽略
- 已选择的选项再次点击会取消选择
- 比较时使用 `normalize_answer`，避免大小写和首尾空格造成重复
- `selections[current_index]` 保存的是用户当前选择的选项文本

## `normalize_answer`

```rust
pub fn normalize_answer(answer: &str) -> String
```

去除首尾空格并转为小写。展示层可用于选择去重，外部验证服务也可以复用同一规则。

## 提交流程

九宫格插件的输出应由宿主组装为：

```rust
pub struct UserSubmission {
    pub cell_id: String,
    pub selected_answers: Vec<String>,
}
```

验证流程位于插件外部：

1. 九宫格插件展示题目并维护 `selections`
2. 用户提交后，宿主读取 `cell_id + selected_answers`
3. 外部验证服务用自己持有的正确答案判定是否通过

本插件不提供 `authorize_open`，也不应该接收正确答案参数。
