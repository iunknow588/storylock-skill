# 使用指南

## 1. 添加依赖

```toml
[dependencies]
storylock-puzzle-plugin = { path = "../../plugins/storylock-puzzle-plugin" }
```

## 2. 准备草稿数据

九宫格插件只需要选项文本：

```json
{
  "nodes": [
    {
      "nodeId": "q1",
      "question": "主角第一次见到谁？",
      "answerOptionsLocalOnly": [
        { "text": "Alice" },
        { "text": "Bob" },
        { "text": "Charlie" }
      ]
    }
  ]
}
```

如果业务草稿中还有 `isCorrect`，本插件也不会读取它。正确答案应由外部验证模块持有。

## 3. 生成九宫格题目

```rust
use storylock_puzzle_plugin::create_open_challenge_from_draft;

let cells = create_open_challenge_from_draft(&draft, 1)?;
```

每个 `StoryLockChallengeCell` 包含：

- `cell_id`
- `prompt_text`
- `answer_options`，固定长度为 9

## 4. 收集用户选择

```rust
use storylock_puzzle_plugin::toggle_selection;

let mut selections = vec![Vec::<String>::new(); cells.len()];

toggle_selection(&cells, &mut selections, 0, 1);
```

再次点击同一选项会取消选择。空选项不会被加入选择结果。

## 5. 提交给外部验证

宿主在用户提交时组装数据：

```rust
let submission = cells
    .iter()
    .zip(selections.iter())
    .map(|(cell, selected_answers)| {
        (cell.cell_id.clone(), selected_answers.clone())
    })
    .collect::<Vec<_>>();
```

然后把 `cell_id + selected_answers` 交给外部验证服务。九宫格插件不调用验证函数，也不接收正确答案。

## 常见问题

### 答案选项为什么固定 9 个？

这是九宫格 UI 的显示约束。不足 9 个时使用空字符串补齐，点击空格不会产生选择。

### 多选顺序重要吗？

插件只收集点击产生的选择列表。是否忽略顺序由外部验证服务决定。

### 可以复用 `normalize_answer` 做验证吗？

可以。它是通用文本归一化工具，但验证逻辑仍应放在插件外。
